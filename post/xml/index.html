XML

XML可扩展标记语言，Extensible Markup Language 。还有一个JSON也可以实现类似的功能，不过更轻量一点。他们大都用于数据的传输，还有很常见的场景是配置文件。
XML是从SGML中扩展出来的。还有很相似的概念是HTML，不过HTML的标签tag都是定死的。XML则没有固定的标记，XML不能直接描述网页具体的外观，内容，它只是描述内容的数据形式和结构。

一个例子，可以看出XML适合人的阅读。self-description but redundent，所以最好用于少量数据的传输。

XML基础

看下面的Demo，包含了最常见的XML要素。

  • prelog：下面的第一行。这是optional的，但是如果有的话必须在第一行。

  • Tag: label for a section of data。

  • Element (元素) : section of data beginning with <tagname> and ending with matching </tagname>。<tagname attr="..."> text... </tagname>

    • 一个Element可以包含的内容有text、attributes、other elements，以及他们的组合。

  • Attribute：Elements can have attributes.属性是name="value"的形式，双引号是必须的。在tag的内部书写。什么时候用属性，什么时候用元素呢？

    • 建议：存储metadata用attribute，比如id。
    • attributes cannot contain multiple values (elements can)
    • attributes cannot contain tree structures (elements can)
    • attributes are not easily expandable (for future changes)

  • comment：下面最后一行。和HTML一样。

特殊字符怎么办

To store string data that may contain tags, without the tags being interpreted as subelements, use CDATA as below

<![CDATA[<course> … </course>]]>

Here, <course> and </course> are treated as just strings. CDATA stands for “character data”.

不想用上面的CDATA，还可以用下面的转义。

< < less than
> > greater than
& & ampersand
' ' apostrophe
" " quotation mark

XML文档模式

为了解析XML，我们最好有一个办法指定XML 的类型，不然XML太灵活了。当然，这不是必须的。

DTD

这是一种办法，用DTD指定XML的格式。如果XML不符合DTD的规定，我们认为这个XML是无效的。

指定Element的子元素，一般有4种类型：

  1. names of elements （第1行，我们用|表示或关系，+表示可以出现1到多次，*表示出现0到多次）
  2. #PCDATA (parsed character data)
  3. EMPTY (no subelements) 。对应于这样的Tag<element></element>或者简单的写成<element />，当然一般都会带属性的，不然要他做什么。
  4. ANY (anything can be a subelement)。如果你在DTD中没有规定，就是ANY。
<!DOCTYPE  university [
        <!ELEMENT university ( (department|course|instructor|teaches)+)>	
        <!ELEMENT department ( dept_name, building, budget)>				要求按照顺序出现
        <!ELEMENT course ( course id, title, dept name, credits)>
        <!ELEMENT instructor (IID, name, dept name, salary)>
        <!ELEMENT teaches (IID, course id)>
        <!ELEMENT dept name( #PCDATA )>
        <!ELEMENT building( #PCDATA )>
        <!ELEMENT budget( #PCDATA )>
        <!ELEMENT course id ( #PCDATA )>
        <!ELEMENT title ( #PCDATA )>
        <!ELEMENT credits( #PCDATA )>
        <!ELEMENT IID( #PCDATA )>
        <!ELEMENT name( #PCDATA )>
        <!ELEMENT salary( #PCDATA )>
]>

指定Attribute的类型

  1. CDATA

  2. ID (identifier) or IDREF (ID reference) or IDREFS (multiple IDREFs)

    IDREFS后面跟一个用空格隔开的reference lists。

可以指定属性存在的必要性：

  1. #REQUIRED必要的，#IMPLIED不是必要的。
  2. 如果是ID，那么在一个文档中不能重复；如果是IDREF，那么一定要在一个文档中别的元素的ID中出现（只要是别的元素的ID就行，DTD不会检查类型）；如果是IDREFS，那么每一个属性都要在别的元素的ID中出现。
<!ATTLIST course
	course_id	ID		#REQUIRED
    dept_name	IDREF	#REQUIRED
	instructors	IDREFS 	#IMPLIED>

XML Schema

DTD比较简单，但是比较简陋。所以我们更多的使用XML Schema。不同于DTD，XML Schema非常复杂。XML Schema 最重要的能力之一就是对数据类型的支持，另外XML Schema本身就是XML。

  • <schema> 元素是每一个 XML Schema 的根元素。需要注意的是，为了避免和用户XML产生冲突，所有的XML Schema标签之前都会加上命名空间xs，比如<xs:schema xmlns:xs="http://reticenceji.github.io/">。

  • <element>限制元素内容的类型。

    • 一个简单的类型：比如<xs:element name="age" type="xs:integer"/>，就限定了这样一个标签<age>28</age>

    • 常用的类型有：xs:string、xs:decimal、xs:integer、xs:boolean、xs:date、xs:time。

    • 一个复杂类型（有属性或子元素）的定义，就需要用到XML的嵌套了。比如下例<xs:complexType> the "note" element is a complex type，

      • 1：<sequence.../>：该元素包含的所有子元素出现顺序与定义顺序必须保持一致；
      • 2：<choice.../>：该元素包含的所有子元素只能出现其中之一；
      • 3：<all.../>：该元素包含的子元素可以任意次序出现。
      • minOccurs定义元素最少出现次数，最小值为0；maxOccurs定义最多出现的次数，最大可以为unbounded，表示不限制出现册次数。
      <xs:element name="note">
    <xs:complexType>
        <xs:sequence>
            <xs:element name="to" type="xs:string"/>
            <xs:element name="from" type="xs:string"/>
            <xs:element name="heading" type="xs:string"/>
            <xs:element name="body" type="xs:string"/>
        </xs:sequence>
        <xs:attribute name="lang" type="xs:string" use="required"/>
    </xs:complexType>
</xs:element>

  • <attribute>限制元素属性。简易元素无法拥有属性。假如某个元素拥有属性，它就会被当作某种复合类型。但是属性本身总是作为简易类型被声明的。比如<xs:attribute name="lang" type="xs:string" use="required"/>

  • 在XML中可以定义Key和Key reference对应SQL中的主键和外键。<field>表示作为key的元素或属性，<selector>表示这个键的作用域。在下面这个例子中，就是说<university>下面的<department>，其dept_name是主键。

    <xs:key name ="deptKey">
    <xs:selector xpath = "/university/department"/>
    <xs:field xpath = "dept_name"/>
</xs:key>

    类似的，这个定义了外键，refer上面的deptKey。

    <xs:keyref name = "courseDeptFKey" refer=“deptKey”>
    <xs:selector xpath = "/university/course"/>
    <xs:field xpath = "dept_name"/>
</xs:keyref>

XML查询和转换

XML是用来传输信息的，我们当然要从XML中提取信息。XPath和XQuery是两种查询的方式。XSLT是XML转换的方式。

XPath🌟

XML文档是一个典型的树型结构，和我们的文件系统是相似的。

XPath使用路径表达式来选取 XML 文档中的节点或者节点集。这些路径表达式和我们在常规的电脑文件系统中看到的表达式非常相似。不过一个文件夹下不会有两个同名文件，但是一个结点下面可能会有两个相同的子节点。在 XPath 中，有七种类型的节点：元素Element、属性Attribute、文本text、命名空间namespace、处理指令processing-instruction、注释comment以及文档（根）节点document node。

XML 文档是被作为节点树来对待的。树的根被称为文档节点document node或者根节点。

XPath是很多应用的基础。

/university-3/instructor/name

常用的表达式：

表达式 描述
nodename 选取此节点的所有子节点。
/ 从根节点选取。
// 从匹配选择的当前节点选择文档中的节点，而不考虑它们的位置。就是说儿子、孙子、曾孙都可以选到
. 选取当前节点。
.. 选取当前节点的父节点。
@ 选取属性。
| 表示或关系，合并多个查询//book/title | //book/price：选取 book 元素的所有 title 和 price 元素。

谓语（Predicates）。谓语用来查找某个特定的节点或者包含某个指定的值的节点。谓语被嵌在方括号[]中。

在下面的表格中，我们列出了带有谓语的一些路径表达式，以及表达式的结果。可以看出那些基本的逻辑运算符和算数运算符都是可以用的。不过是=不是==哈：

路径表达式 结果
/bookstore/book[1] 选取属于 bookstore 子元素的第一个 book 元素。
/bookstore/book[last()] 选取属于 bookstore 子元素的最后一个 book 元素。
/bookstore/book[last() - 1] 选取属于 bookstore 子元素的倒数第二个 book 元素。
/bookstore/book[position() < 3] 选取最前面的两个属于 bookstore 元素的子元素的 book 元素。
//title[@lang] 选取所有拥有名为 lang 的属性的 title 元素。
//title[@lang='eng'] 选取所有 title 元素，且这些元素拥有值为 eng 的 lang 属性。
/bookstore/book[price > 35.00] 选取 bookstore 元素的所有 book 元素，且其中的 price 元素的值须大于 35.00。
/bookstore/book[price > 35.00]/title 选取 bookstore 元素中的 book 元素的所有 title 元素，且其中的 price 元素的值须大于 35.00。

通配符*：

路径表达式 结果
/bookstore/* 选取 bookstore 元素的所有子元素。
//* 选取文档中的所有元素。
//title[@*] 选取所有带有属性的 title 元素。

XQuery

这个比XPath强大，但是也更复杂。是基于XPath的。感觉这玩意儿有点像shell语言。

FLWOR

一个基本的XQuery是由FLWOR组成的：

for $x in /university/course
let $courseld := $x/@course_id
where $x/credits > 3
return <course_id> {$courseld} </course_id>

最基本的我们一般只需要for+return

  • for $x in [xpath查询]，然后就会循环枚举每一个匹配到的项目。有点像select
  • return字句可以对查询内容加工构造XML形式的结果，不过需要给变量外面套{}

其他的：

  • let可以用来加工查询结果，提取有效信息，然后赋值给变量。

  • where可以对查询结果进行筛选。

  • order by，显然是用来对查询结果排序的

高级特性

XQuery 含有超过 100 个内建的函数。这些函数可用于字符串值、数值、日期以及时间比较、节点和 QName 操作、序列操作、逻辑值等等。自定义函数：

XSLT

据说这个不怎么考。了解以下即可。

XSL = XML 样式表。XML 不使用预先定义的标签（我们可以使用任何喜欢的标签名），并且这些标签的意义并不都那么容易被理解。元素意味着一个 HTML 表格，一件家具，或是别的什么东西 - 浏览器不清楚如何显示它。XSL 可描述如何来显示 XML 文档！

XSL 包括三部分：

  • XSLT：一种用于转换 XML 文档的语言。
  • XPath：一种用于在 XML 文档中导航的语言。
  • XSL-FO：一种用于格式化 XML 文档的语言。

这个是用来转换的。什么意思呢？比如从XML转换成HTML，或者从一个XML转换成另一个XML。

XSLT Templates，也是一个XML文档。

<xsl:template match="/bank-2/customer">
    <custom>
        <xsl:value-of select="customer_name"/>
    </custom>
</xsl:template>
<xsl:template match="*"/>

xsl:template ：用XPath来匹配一些节点

xsl:value-of ：标签的内容
