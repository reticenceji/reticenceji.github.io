ADS-RB Tree&B+ Tree

2021-03-08 5 min read # DS

根据对平衡的不同定义，我们还有红黑树和B+树。

Red-Black Tree


Target：加速动态搜索的平衡树。
定义：

A red-black tree is a binary search tree that satisfies the following red-black properties:
(1) Every node is either red or black.
(2) The root is black.
(3) Every leaf (NIL) is black.
(4) If a node is red, then both its children are black.(平衡)
(5) For each node, all simple paths from the node to descendant leaves contain the same number of black nodes.(平衡条件)
一些概念：

  • black-height：The black-height of any node x, denoted by bh(x), is the number of black nodes on any simple path from x (x not included) down to a leaf. bh(Tree) = bh(root).
  • internal node：可以认为是非NULL结点。
  • external node：NULL结点。

操作

Rotate：左旋和右旋是基本的旋转操作，和AVL Tree中类似。

Insert：红黑树的插入操作最多有2次旋转。
我们把插入的点（4）染成红色，然后考虑：

  1. 如果parent是黑色，那么插入成功。
  2. 如果parent是红色：
    1. 如果parent的cousin是红色，执行case1的操作。不过要注意的是，这个操作可能会带来新的问题，看到顶部的结点变成红色了。(如果顶部结点变红了，再把他变成黑的)
    2. 如果parent的cousin是黑色：
      1. 如果是RL型或LR型，执行case2的操作，转换成case3。
      2. 如果是RR型或LL型，执行case3的操作。
insert_fixup(T, z)
{
    while (z->parent->color == RED) //如果是黑色就完成了
    {
        if (z->parent == z->parent->parent->left)
        {
            y = z->parent->parent->right;
            if (y->color ==RED)     //case 1
            {
                z->parent->color = BLACK;
                y->color = BLACK;
                z->parent->parent->color = RED;
                z = z->parent->parent;
            }
            else 
            {
                if (z == z->parent->right)     //case 2
                {
                    z = z->parent;
                    left_rotate(T,z)
                }
                z->parent.color = BLACK;            //case 3
                z->parent->parent->color = RED;
                right_rotate(T,z->parent->parent);
            }
            else 
            }
        }    
    }
}

Delete：红黑树的删除操作最多有3次旋转，麻烦的不行。参考链接
回顾一个普通的二叉树的删除过程

  • 0度节点：Reset its parent link to NULL.很简单，删了就好。
  • 1度节点：Replace the node by its single child.也比较简单，把它的孩子当作它。
  • 2度节点：Replace the node by the largest one in its left subtree or the smallest one in its right subtree. Then Delete the replacing node from the subtree.

所以我们看到最简单的操作是删掉一个red leaf。所以删除的解决办法保证从上到下删除期间树叶是红的。
在整个讨论中，令X为当前结点，T是他的sbling，P是他的parent。如果X就是我们要删除的结点，我们把他和X左子树上最大的结点或右子树上最小的结点交换。
当我们沿着树向下遍历的时候，我们保证X是红色的。我们首先把root染成红色（最后再染回黑色，也可以忽略它，我不是很明白这个的意义是什么）。当我们到达一个新的结点的时候，我们就知道P是红的（就是之前的X），并且X和T是黑的（不能有两个连续的红色）。

B+ Tree

B+树不是一个二叉树，但是是搜索树。适用于超大规模的数，迎合硬件存储机制。

A B+ tree of order MMM is a tree with the following structural properties:
(1) The root is either a leaf or has between 222 and MMM children.
(2) All nonleaf nodes (except the root) have between ⌈M/2⌉\lceil M/2\rceil⌈M/2⌉ and MMM children.
(3) All leaves are at the same depth.
