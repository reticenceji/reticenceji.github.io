什么是操作系统

  • Resource abstractor and a resource allocator
  • Software layer between the applications and the hardware because the hardware would be too difficult for users to use

操作系统的启动

操作系统的启动，就是把操作系统的代码（数据）从Disk加载到内存中，并开始执行。这个工作是被一个称为BootLoader的程序完成的。
加载完成之后，操作系统就开始运行了。
Kernel不是一个正在运行的job。

俯瞰

Kernel里面是没有什么内存保护的，kernel自己负责处理内存错误，所以编程的时候需要万分小心。

Kernel可以执行privileged指令（准确来说，这是硬件架构赋予Kernel的权利，我们让Kernel运行在硬件架构的Privileged模式，让用户运行在硬件架构的User模式。现在的架构至少支持两个模式，古老的Intel 8086就只有一个模式，所以我们可以随意修改DOS的代码、随意修改中断向量、直接执行IO指令），这样Kernel就可以把IO、内存管理、特殊寄存器统统封装起来防止用户篡改了。

用户需要调用操作系统的某些功能（或者说用户程序本来也可以自己完成，但是因为有了操作系统所以必须通过操作系统来完成），所以我们有中断和异常，可以把他们当作从User到Kernel的入口Entry。

操作系统主要提供的服务，以及我们在Lab中的简单实现：

  • Process Management：通过time interrupt来调度，通过system call来fork、exec、exit等。联想进程图状态切换。还有进程之间的同步、通信。
  • Memory Management。非常宏大，一个是执行用户程序，要把用户程序装载到物理内存上，还有可能需要为他动态的分配空间，还要让他可以访问到Kernel（指执行）。中间有分配的算法。
  • Storage Management：这个也属于IO的一部分。不过我们用文件系统来把Storage抽象成一系列接口，然后用虚拟文件系统VFS再把文件系统抽象成统一的接口。利用这些接口管理Storage。物理内存上的内容被我们抽象成为文件。
  • I/O Management：每一个硬件都有不同的处理方式，我们需要驱动程序来直接处理硬件，同时也要把他们抽象成文件来统一处理。
  • Protection and Security：这又是另一个故事了。
