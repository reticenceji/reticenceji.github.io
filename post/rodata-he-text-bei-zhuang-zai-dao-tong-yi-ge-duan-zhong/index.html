.rodata和.text被装载到同一个段中

2020-11-30 2 min read # Others


所以有三个问题：

  1. .rodata section里面的内容是可执行的？（虽然好像不会带来什么安全问题）
  2. proc/pid/maps里面看到的r--p段是什么东西？
  3. 这个东西难道和编译选项有关系？

.rodata section里面的内容是可执行的

做个试验：

int main(void)
{
    //>>> asm("next: jmp next")
    //b'\xeb\xfe'
    ((void(*)(void))"\xEB\xFE")();
}

然后程序确实死循环了。用GDB调试一下也行。

proc/pid/maps里面看到的r--p段是什么东西

有可能是上面那个 INTERP，对应与.interp

这个东西难道和编译选项有关系

可能是因为程序太小了，为了节省空间GCC 把他们编译到一个section了。
用lds控制一下对齐，让rodata对齐到一个页上。我们发现rodata section被映射到了一个只读的segment里了。
看来确实是为了节省空间考虑。
我们再执行一下上面的代码，果然段错误了。
附上lds:

int main(void)
{
    //>>> asm("next: jmp next")
    //b'\xeb\xfe'
    ((void(*)(void))"\xEB\xFE")();
}
