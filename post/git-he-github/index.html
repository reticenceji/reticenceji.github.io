需求一：如何在本地创建一个仓库，然后和远程联动？

  1. 创建公钥： ssh-keygen -t rsa -C "your_email@example.com"
    然后查看公钥：cat ~/.ssh/id_rsa.pub，复制其中的内容，添加到Github的SSH中，名字随意。

    然后相当于添加了一个身份认证。在本地填上自己的email和name，就可以和远程仓库联动了。

  2. 创建文件夹，git init

  3. 添加Origin远程仓库：git remote add origin git@github.com:reticenceji/oslab.git(注意不要添加Https开头的那个远程仓库的地址，选SSH的)。添加后，远程库的名字就是origin，这是Git默认的叫法，也可以改成别的，但是origin这个名字一看就知道是远程库。

  4. 如果远程仓库是空的，你需要初始化一下（也就是第一次上传时）git push --set-upstream origin master

  • 2,3,4是你自己创建库，如果是已经存在的库，直接git clone就好了。
  • git remote -v 查看origin
  1. 查看信息
    git remote -v     #查看远程仓库信息
git config --global -l   #列出本地个人信息，建议把email和github设成一样

需求二：上传、下载的基本操作

  1. 关于上传的基本操作
git status     #查看当前工作区的状态
git add filename1,filename2	#保存到暂存区stage
git add .	#保存全部文件到stage
git commit -m "提交说明"	#保存到repository，提交说明尽可能的详细。
git push -u origin master	#第一次提交到远程仓库
git push origin master	#之后提交到远程仓库
  1. 关于回退的基本操作
git log		#查看提交历史，以便确定要回退到哪个版本。
git reflog	#查看命令历史，以便确定要回到未来的哪个版本。
git reset --hard HEAD^	#返回到上一个版本

3 git pull origin从远程拉取最新版本。

需求三：本地代码和远程代码有冲突的时候，pull失败/push失败

当本地代码和远程代码有冲突的时候，执行git pull操作的时候，会提示有冲突，然后直接终止本次pull。

以下push为例子。我在本地和远程分别编辑了README导致了冲突。

这个有问题高级的方式解决，但是比较繁琐。建议在别的地方备份一下，然后强制合并：

   git fetch --all      #从远程下载最新的，而不尝试合并或rebase任何东西。
   git reset --hard origin/master   #将主分支重置为您刚刚获取的内容

然后你手动观察一下哪里不太一样，手动merge再push。

  • 这个问题最好避免。避免方法为在写之前先pull一下更新到最新版本，写完之后add-commit-push三连直接上传。其实pull所做的操作就是fetch+merge，push也是有merge；所以不要你在写的时候正好别人也在写同样的文件提交了，一般不会出问题。
  • 强制pull(慎用)，加个-f选项
  • origin可以认为是指向某个repository的指针，master是branch名字（主分支）。origin/master就是那个远程repo的主分支。我们输入的git push origin master是git push origin master:master的简写实际上是把本地的master推送到远程的master

需求四：不用命令行使用git

建议使用VSCode.

  • 他会列出所有的Changes,点击Changes旁边的+号保存changes到stage，相当于add。
  • 最上方的✔️是commit

需求五：版本的推进

使用git的tag功能
方法一：使用github
可以在github里添加tag
然后git pull origin #不能 git pull origin master，在本地就会有tag了。

方法二：使用git
创建一个标签：git tag -a v1.0 -m "lab1的最终版本"
列出所有的标签：git tag

然后如果你想返回到之前的版本：git checkout v1.0，然后你可以查看当时的成果。不过你想做出修改，也不会改到现在的版本。你先应该git checkout master回到当前的主分支，然后继续工作。

需求六：commit之后后悔了，想要撤回

执行git reset --soft HEAD^，就是回到上一个版本。你的代码还在的。

需求六：想做一个小改动，覆盖上一次的提交

本地覆盖记录
git commit --amend -m "xxxx"
线上覆盖提交，-f 表示强制push,远程的最新的一次commit被删除
git push origin 分支名称:分支名称 -f

需求七：查看版本之间的区别

git diff命令可以查看区别，主要使用如下：
git diff查看当前没有add的内容修改。(这个vscode可以直接显示)
git diff --cached查看已经add没有commit的修改。
git diff 版本号1 版本号2查看版本之间的区别，举个例子，比如我刚刚pull了一个版本，我想查看他和上一个版本的区别，可以用git diff HEAD^ HEAD。

需求八：创建开发分支

git branch name可以创建分支，比如创建develop分支。
创建完之后，用git checkout name切换分支。
之后我们的commit操作就会提交到我们当前的这个分支上了。

之后，当我们做了一定的工作然后想要合并回主分支的时候，用merge或者rebase。merge是最简单的，永远不要rebase那些已经推送到公共仓库的更新。
