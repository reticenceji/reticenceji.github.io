拓扑排序

Topological sorts.适用于不循环(acyclic)的图。不过也可以通过拓扑排序判断图是否循环。拓扑排序的结果不一定是唯一的。

拓扑排序的思想是

  1. 初始化一个入度为0的点的队列（队列或堆栈）。
  2. 如果队列为空，那么结束。
  3. 从队列中出队vertex V，并给V的后继indegree-1。
  4. 寻找入度为0的点的入队。
    如果最后不存在入度为0的节点，那就说明有环，无解。
Topsort(Graph G)
{
	Queue Q;	//queue的操作就当已经实现
    int inDegree[G.Nv];		//入度数组 
    int Count = 0;
    Vertex V,W;
    
    Initialize(inDegree,G);		//假装初始化好了入度数组
    for (int i=0;i<G.Nv;i++)
        if (inDegree[i]==0) Enqueue(Q,G.G[i]);
    while (!empty(Q))
    {
        V = Dequeue(Q);
        print(V);       //反正标记到他了
        for i in V.next:		//开始瞎写了
        	Enqueue(Q,i);
    }
}

AOE network

用顶点表示事件，用边表示活动，权值表示活动的持续时间。
只有在某顶点代表的事件发生后，从该顶点出发的各活动才能开始。只有在进入某顶点的各活动都已经结束，该顶点代表的事件才能发生。
因此在 AOE 网中，只有一个入度为 0 的点表示工程的开始，即源点，也只有一个出度为 0 的点表示工程的结束，即汇点。是无环图。

关键路径的计算

在 AOE 网上，从源点到汇点的具最大路径长度（该路径上行各活动持续时间的和）的路径称为关键路径，关键路径上的活动称为关键活动。
有两个概念，一个是最早开始时间(EC)，最晚开始时间(LC).对于关键路径上的事件，EC==LC。

  • 可以用类似于拓扑排序的方法求最早开始时间：EC[v] = Max{v前继的EC}+self（所以要记录每个点的最早开始时间）。

Dijkstra

Dijkstra 是单源最短路算法，不能处理带负边权的情况，用邻接矩阵或邻接表存图。是一种贪心算法。
这个链接可以图解。
unkown优先队列：把未曾经过的点组成一个最小堆。这样对稀疏图高，否则要遍历找最小。

初始化一个distance数组，标记起点为0,其他的点为INF。初始化一个unknown优先队列，把起点入队。
从unknown中pop出一个vertex,遍历vertex的邻点：
	如果邻点known(就是从unknown队列中pop出来过，可以存储一下)，则不做操作
	否则
		判断从该vertex出发到它邻点的距离，是否能更新distance数组。
		将邻点入unknown优先队列（有的点可能被重复入队，其实是要改变他的位置）
如果unknown空了，就结束了

最大流

最大流问题：在一个单源点、单汇点的网络流中找到一条最大的可行流。

有向图称为网络。顶点称为节点（Node）而边称为弧（Arc）。一道流必須符合一個节点的进出的流量相同的限制，除非這是一個源（Source）──有较多向外的流，或是一個汇（Sink）──有较多向內的流。

在容量网络中满足以下条件的网络流称为可行流:

  • 容量限制（Capacity Constraints）: .
  • 流守恒（Flow Conservation）: 除非 或 ，否则 。一结点的净流是零，除了“制造”流的源点和“消耗”流的汇点。

最大流就是最大的可行流。

Edmonds–Karp算法

时间复杂度为O(VE^2)。当我实际编程的时候，有这些感悟：

  1. 因为涉及查找，所以最好用邻接表而不是邻接矩阵
  2. 难点在于增广路的查找和存储。
initMaxFlow
while there is an augmenting path
    find an augmenting path using BFS
    for each edge u->v in the path
        decrease capacity u->v by bottleneck
        increase capacity v->u by bottleneck
    increase maxflow by bottleneck

功能：用广度优先搜索查找增广路
参数：source和destination指定源和汇，grpah->matrix是邻接矩阵，结果存储在ADT stack中。下面是伪代码：

int find_argument(int source,int destination, Lgraph* graph,Stack* stack)
{
    //初始化，visited表示是否已经访问过
    for (i=1;i<=graph->Nv;i++)
        visited[i] = 0;
    visited[source] = 1;
  
    i = 1;
    while (1)
    {
         //没有访问过并且有路，就是下一个目标
        if (graph->matrix[source][i]>0 && visited[i]==0)       
        {
           stack.push( (source,i) )
            visited[i] = 1;
            if (i==destination) return True;   
            source = i;
            i=1;
            continue;
        }
        i++;   
        // 说明没找到
        if (i>graph->Nv) 
        {
            if (stack.isEmpty() ) return false; //如果栈空了说明没有路，否则退回上一步
            source,i = stack.pop()
            visited[i] = 0;
            i++;
        }
    }
}

最小生成树 Minimum spanning tree

Minimum spanning tree。连通图都有最小生成树。找n-1条边把图联通，让边之和最小。
参考链接知乎。Prim算法以点为中心，一个点一个点的扩展，和Dijkstra算法非常相似。Kruskal算法是并查集。

Prim算法

1）以某一个点开始，寻找当前该点可以访问的所有的边；
2）在已经寻找的边中发现最小边^1，这个边必须有一个点还没有访问过，将还没有访问的点加入我们的集合，记录添加的边；
3）寻找当前集合可以访问的所有边，重复2的过程，直到没有新的点可以加入；
4）此时由所有边构成的树即为最小生成树。

  1. 发现最小边：最好用堆
  2. 标记没有访问的点：可以用一个数组

Kruskal 算法

时间复杂度貌似取决于排序算法。

现在我们假设一个图有m个节点，n条边。
首先，我们需要把m个节点看成m个独立的生成树，并且把n条边按照从小到大的数据进行排列。
在n条边中，我们依次取出其中的每一条边，如果发现边的两个节点分别位于两棵树上【2】，那么把两棵树合并成为一颗树【3】；如果树的两个节点位于同一棵树上，那么忽略这条边，继续运行。
等到所有的边都遍历结束之后【4】，如果所有的生成树可以合并成一条生成树【1】，那么它就是我们需要寻找的最小生成树，反之则没有最小生成树。

  1. 是否合并成一棵树：利用并查集，看有几个集合。
  2. 两个节点分别位于两棵树上：并查集Find
  3. 把两棵树合并成为一颗树：并查集Union，如果需要的话要记录连接的边。
  4. 如果找到Nv-1条边也可以提前结束

图的遍历

可以找一个图的所有生成树（没必要最小），然后遍历树。

//深度优先搜索。最好用邻接表，这样时间复杂度是`T(E+V)`
void DFS(Vertex V)
{
	visited[V] = true;
    /* 要做点事情的吧 */
	for (each W adjacent to V)
		if (!visited[W])
			DFS(W);
}
void BFS(Vertex V)
{
    visited[V] = true; Enqueue(V);
    while (NotEnpty())
    {
        Dequeue(V);
         /* 要做点事情的吧 */
        for (each W adjacent to V)
			if (!visited[W])
				Enqueue(W);
    }
}
//如果是连通图，那么找一个点DFS就够了。如果不是，遍历所有的树
void (Graph G)
{
    for (each V in G)
        if (!visited[V])
            DFS(V);
}

双连通图 Biconnection

点双连通：在无向连通图中，如果删除该图的任何一个结点都不能改变该图的连通性，则该图为双连通的无向图。
类似的有边双连通：在无向连通图中，如果删除该图的任何一个边都不能改变该图的连通性，则该图为双连通的无向图。
割点：在一个无向图中，如果有一个顶点集合，删除这个顶点集合以及这个集合中所有顶点相关联的边以后，图的连通分量增多，就称这个点集为割点集合。如果割点集合只有一个点，那么就叫割点/断点(cut/articulation point)。

Tarjan算法

Tarjan算法是是Tarjan发明的算法的统称。一般来说指下面求解下面三个问题的算法

  1. 有向图的强连通分量。
  2. 无向图的双联通分量。下面讲这个。
  3. 最近公共祖先问题。

假设我们有一个连通的无向图G。
首先，用DFS获得G的生成树spinning tree。
然后寻找图G的articulation point，具体方法如下：

  • root是割点 等价于 他有两个children
  • 其他的点是割点 等价于 他至少有一个孩子，并且他不能通过它的孩子跳到它的祖先。

实现：
用DFS获得图G的生成树，根据访问顺序我们给每一个节点v一个Num[v]Num[v]Num[v]，也就是生成树的preorder遍历顺序。
然后我们再维护一个Low[u]=min(Num[u],min(Low[w]∣w is a child of u),min(Num[w]∣(u,w)is a back edge))Low[u] = min( Num[u], min( Low[w] | w\ is\ a\ child\ of\ u), min(Num[w] | (u,w) is\ a\ back\ edge))Low[u]=min(Num[u],min(Low[w]∣w is a child of u),min(Num[w]∣(u,w)is a back edge))，Low[u]Low[u]Low[u]的意思是u可以通过它的孩子、它的背向边back edge到达的最小的点。
那么图G的articulation point v，满足：

  • v=root and v have 2 child。
  • v!=root and v至少有一个孩子w满足num[v]≤low[w]num[v] \le low[w]num[v]≤low[w]。

强连通

强连通：在有向图G中，如果两个顶点vi,vj间（vi>vj）有一条从vi到vj的有向路径，同时还有一条从vj到vi的有向路径，则称两个顶点强连通(strongly connected)。
强连通图：如果有向图G的每两个顶点都强连通，称G是一个强连通图。
强连通分量：有向图的极大强连通子图，称为强连通分量(strongly connected components)，也就是说是图G的一些特殊子集。

强连通性也是一种顶点之间等价关系，因为它有着以下性质。

  • 自反性：任意顶点v和自己都是强连通的。
  • 对称性：如果v和w是强连通的，那么w和v也是强连通的。
  • 传递性：如果v和w是强连通的，且w和x也是强连通的，那么v和x也是强连通的。

Korasaju算法

我们可以通过执行两次DFS找出图的strongly connected components。

第一步：拿到当前图G的反向图G-。（转置）
第二步：对G-进行深度优先遍历(DFS)，拿到G-的reversePost（G的伪拓扑排序）。
第三步：根据reversePost中节点的顺序，对当前图G进行深度优先遍历（DFS），即可得到强连通分量的数量。

首先，G-和G的强连通分量是一致的。
本来I可能会遍历到E的，但是通过reversePost，我们让E先于I被访问。对于这种单向连通的情况，我们让儿子先于父亲被访问了，就不会被访问了。

int DFS1(int  v) /* visitedthevnode */
{
    visited[v] = 1;
    for (int i = 1;i <= N;i++)
        if (!visited[i] && maxtrx[i][v])    //注意这里是矩阵的转置，所以是v,i。如果你觉得这样locality很差的话也可以新建一个matrix
            DFS1( i );
    S.push( v );
    return 0;
}
 
int DFS2( int v )
{
    visited[v] = 1;
    for (int i = 1;i <= N;i++)
        if ( !visited[i] && nmap[v][i] )
            DFS2( i );
    /* 在同一次遍历到的节点就在同一个强连通分量了 
     * 你可以把他们存下来或者做别的操作 
     */
    return 0;
}

void korasaju()
{
    for (i=0;i<N;i++)
        if (!visited[i]) DFS(i);        //通过第一次DFS得到一个Stack，由此得到reversePost
    visited = {0};
    for (i=0;i<N;i++){
        t = S.pop();
        if (!visited[t]) DFS(t);
    }  
}

欧拉图

如果存在一条路经过图上的每条边edge有且仅有一次，那么称这条路是欧拉路，如果这条路是回路，那么称作欧拉回路。

  • 如果一个图上的节点vertex的度degree都是偶数even或有且仅有两个奇数odd，那么我们可以找到一条欧拉路Euler path。
  • 如果一个图上每个vertex的degree都是偶数，那么我们可以找到一个欧拉回路Euler circuit。

最大独立集

独立集就是，就是一个点集，点集中的各点没有关系。
最大独立集就是，点的个数最多的独立集。

最大独立集 == 点的总数 - 最小点覆盖。
这是为什么呢？

简单证明：
最大独立集合的定义 是 最大无关系点的集合。

而最小点覆盖的定义是，二分图中每个边至少一个端点在该点集中 的 最小点集。

如果去掉这些点， 相应的关系（边）也都没有了。剩下的点之间就相互没有关系，变成了独立集。

因为去掉的是 最少的点的集合， 所以剩下的就是最大独立集合。

最大独立集 就可以通过匈牙利算法来做。
