Flask概述

Flask是一个用Python编写的Web应用程序框架。 它由 Armin Ronacher 开发，他领导一个名为Pocco的国际Python爱好者团队。 Flask基于Werkzeug WSGI工具包和Jinja2模板引擎。两者都是Pocco项目。

  • Web Application Framework（Web应用程序框架）或简单的Web Framework（Web框架）表示一个库和模块的集合，使Web应用程序开发人员能够编写应用程序，而不必担心协议，线程管理等低级细节。

  • WSGI（Web Server Gateway Interface，Web服务器网关接口）已被用作Python Web应用程序开发的标准。 WSGI是Web服务器和Web应用程序之间通用接口的规范。

  • Werkzeug是一个WSGI工具包，它实现了请求，响应对象和实用函数。 这使得能够在其上构建web框架。 Flask框架使用Werkzeug作为其基础之一。

  • jinja2是Python的一个流行的模板引擎。Web模板系统将模板与特定数据源组合以呈现动态网页。

环境搭建

  • 我们利用virtualenv搭建一个独立的python环境。首先安装virtualenv：
    sudo apt-get install python-virtualenv

  • 然后创建一个本地环境，切换到你的工作目录。更多选项可以help：
    virtualenv -p python3 venv

  • 激活venv环境和取消激活：
    source venv/bin/activate和deactivate

  • 然后利用pip安装flask。不知道为什么我需要翻墙才行，我的源应该就是国内的。
    pip install flask

  • 建一个git仓库，如果不想把环境也带进去的话就编辑.gitignore文件
    git init
    修改.gitignore，添加/venv/

如果你能正常的import flask，环境就搭建完成了。另外的，推荐使用vscode。当然能用专业版的Pycharm就更香了。

Demo

from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
    return "<h1>Hello world</h1>"

if __name__=='__main__':
    app.run()

app.run(host, port, debug, options)表示运行应用程序，所有的参数都是可选的。

一些概念：

  • 上面的index函数称为视图函数view function。
  • index函数的返回值称为响应response。
  • 处理URL和函数之间关系的程序称为路由。我们把/和index函数绑定在了一起。

字段构造函数的第一个参数是把表单渲染成 HTML 时使用的标号。可选参数 validators 指定一个由验证函数组成的列表。

from flask_wtf import Form
from wtforms import StringField, SubmitField
from wtforms.validators import Required

#创建一个Form类
class CardForm(Form):
    text = StringField("内容",validators=[Required()])
    submit = SubmitField("上传")
   
#表单的上传多以POST方式
@app.route('/',methods=['GET','POST'])
def index():
    form = CardForm()
    if form.validate_on_submit():
        session['text'] = form.text.data
        return redirect(url_for('index'))
    return render_template("index.html",form=form,text = session.get('text'))

HTTP

最基本的，我们的Web服务器要对来自客户的request做出response。我们来回顾一下request的报文格式，我随便截取了一个请求报文：

GET / HTTP/1.1
Host: 127.0.0.1:5000
Connection: keep-alive
Upgrade-Insecure-Requests: 1
User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/88.0.4324.96 Safari/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9
Sec-Fetch-Site: none
Sec-Fetch-Mode: navigate

Accept-Encoding: gzip, deflate, br
Accept-Language: en-US,en;q=0.9,zh-CN;q=0.8,zh;q=0.7

request body
  • 最基本的，我们要对请求的URLhttp://127.0.0.1:5000/和对应的请求方法GET做出response。
  • 进一步的，我们可能需要针对请求头的某些字段做出response。
  • 进一步的，我们可能需要对请求体做出response。

这是一个简单的例子。事实上在Flask中为我们提供了response和request对象，让我们能更好的控制返回和解析请求。

关于 response

简单的demo中，对http://127.0.0.1:5000/和对应的请求方法GET（默认）做出响应，我们的返回值是一段简单的文本，会被转换成response返回。

视图函数的返回值会自动转换为一个响应对象。如果返回值是一个字符串，那么会被 转换为一个包含作为响应体的字符串、一个 200 OK 出错代码 和一个 text/html 类型的响应对象。如果返回值是一个字典，那么会调用 jsonify() 来产生一个响应。以下是转换的规则：

  1. 如果视图返回的是一个响应对象，那么就直接返回它。
  2. 如果返回的是一个字符串，那么根据这个字符串和缺省参数生成一个用于返回的 响应对象。
  3. 如果返回的是一个字典，那么调用 jsonify 创建一个响应对象。
  4. 如果返回的是一个元组，那么元组中的项目可以提供额外的信息。元组中必须至少 包含一个项目，且项目应当由 (response, status) 、 (response, headers) 或者 (response, status, headers) 组成。 status 的值会重载状态代码， headers 是一个由额外头部值组成的列表 或字典。
  5. 如果以上都不是，那么 Flask 会假定返回值是一个有效的 WSGI 应用并把它转换为 一个响应对象。

如果想要在视图内部掌控响应对象的结果，那么可以使用 make_response() 函数。

关于 request

请求对象在 API 一节中有详细说明这里不细谈（参见 Request ）。 这里简略地谈一下最常见的操作。首先，你必须从 flask 模块导入请求对象:

from flask import request

对于每一个请求，Flask会构建请求上下文。你不用操心并发的请求。

Doc中的demo：通过使用 method 属性可以操作当前请求方法，通过使用 form 属性处理表单数据（在 POST 或者 PUT 请求 中传输的数据）。以下是使用上述两个属性的例子:

@app.route('/login', methods=['POST', 'GET'])
def login():
    error = None
    if request.method == 'POST':
        if valid_login(request.form['username'],
                       request.form['password']):
            return log_the_user_in(request.form['username'])
        else:
            error = 'Invalid username/password'
    # the code below is executed if the request method
    # was GET or the credentials were invalid
    return render_template('login.html', error=error)

数据库⭐️

Flask的可扩展性可以让你选择多种处理数据库的方式。这里只介绍一种。flask-sqlalchemy

pip install flask-sqlalchemy，SQLAlchemy 提供了高层 ORM（ORM框架的作用就是把数据库表的一行记录与一个对象互相做自动转换），也提供了使用数据库原生 SQL 的低层功能。

Demo

from flask_sqlalchemy import SQLAlchemy
basedir = os.path.abspath(os.path.dirname(__file__))
app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] ='sqlite:///' + os.path.join(basedir, 'data.sqlite')
#每次请求结束后都会自动提交数据库中的变动。
app.config['SQLALCHEMY_COMMIT_ON_TEARDOWN'] = True
db = SQLAlchemy(app)

连接数据库

在这些 URL 中，hostname 表示 MySQL 服务所在的主机，可以是本地主机（localhost），
也可以是远程服务器。数据库服务器上可以托管多个数据库，因此 database 表示要使用的数据库名。如果数据库需要进行认证，username 和 password 表示数据库用户密令。另外，如果连接的是PostgreSQL的话，需要pip install psycopg2（可以认为psycopg2是连接PostgreSQL的基础，在SQLAlchemy里面会调用）：

  • MySQL mysql://username:password@hostname/database
  • Postgres postgresql://username:password@hostname/database
  • SQLite（Unix） sqlite:////absolute/path/to/database
  • SQLite（Windows） sqlite:///c:/absolute/path/to/database

创建数据表，我们需要首先规定好数据库的模型，你可以认为这规定了一张表的结构。__tablename__规定了表的名字，里面的每一个attrbute规定了表的列：

#注意这里的db就是上面的 db = SQLAlchemy(app)
#db 表示程序使用的数据库，同时还获得了 Flask-SQLAlchemy提供的所有功能。
class Card(db.Model):
    __tablename__ = 'cards'
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(64), unique=True)
    text = db.Column(db.Text)
    time = db.Column(db.DateTime)
    good = db.Column(db.Integer)
    
    def __repr__(self):
        return f"<Card {self.name}>"
    
 # 创建当前应用中声明的所有模型类对应的数据表，db.drop_all()是删除表
db.create_all()

增删改查

增删：首先要知道的是会话session。这个是数据库中事务的增强概念。

#假设这里的User是我之前创建的模型，就像Card一样
>>> me = User('admin', 'admin@example.com')
#增
>>> db.session.add(me)
>>> db.session.add_all([admin_role, mod_role, user_role,user_john, user_susan, user_david])
#直到commit才真正做了修改
>>> db.session.commit()
#删
>>> db.session.delete(me)
>>> db.session.commit()

查：Flask-SQLAlchemy 为每个模型类都提供了 query 对象。最基本的模型查询是取回对应表中的所有记录。下面的查询假设数据库中有如下条目：

id username email
1 admin admin@example.com
2 peter peter@example.org
3 guest guest@example.com

通过用户名查询用户，使用filter_by()作为过滤器表User中的记录，指定first()返回查询到的第一个结果:

>>> peter = User.query.filter_by(username='peter').first()
>>> peter.id
1
>>> peter.email
u'peter@example.org'

同上但是查询一个不存在的用户名返回 None:

>>> missing = User.query.filter_by(username='missing').first()
>>> missing is None
True

使用更复杂的表达式查询一些用户:

>>> User.query.filter(User.email.endswith('@example.com')).all()
[<User u'admin'>, <User u'guest'>]

按某种规则对用户排序:

>>> User.query.order_by(User.username)
[<User u'admin'>, <User u'guest'>, <User u'peter'>]

限制返回用户的数量:

>>> User.query.limit(1).all()
[<User u'admin'>]

用主键查询用户:

>>> User.query.get(1)
<User u'admin'>

总结

至此，基础知识就全部具备了。不过在具体开发的过程中，需要注意的还有工程的管理。

做了一个初级的Demo，非常的垃圾。不过还是学到了一些东西。主要是对Web的工作有了更深一点的认识。

  1. 在底层看，无非是Socket的通信罢了。
  2. 在应用层看，是Request和Response的交互。对用户的每一个Request，我都要给出对应的Response。
  3. 在开发的过程中看，分为前端和后端
    1. 前端，要做好页面设计，给用户一个良好的感官体验。
    2. 前端，要向后端发送表单。要设计好让用户填写的表单，通过POST请求发送到后端。
    3. 后端，要对用户的请求做出相应的响应。在请求中最主要的是关注URL，如果是POST请求的话还有Request Body。如果是登录的话可能还有Cookie。所以我们要根据这些参数构造不同的视图函数，做出对应的回复。
    4. 后端，要操纵数据库，从数据库中获取相应的信息返回给用户。
