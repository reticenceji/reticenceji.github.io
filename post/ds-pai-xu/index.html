高级排序算法

有很多排序算法。从C库函数qsort可以看出

  1. 我们需要知道排序的元素个数。
  2. 我们要保证两个元素之间是可以比较的，必然存在一个大小关系。

可视化排序算法。我们没有讲冒泡排序、选择排序。
基于比较的排序，最好的时间复杂度也是O(NlogN)O(N log N)O(NlogN)

Insertion Sort

比如我们整理扑克牌。

void InsertionSort ( ElementType A[ ], int N ) 
{ 
    int  j, P; 
    ElementType  Tmp; 

    for ( P = 1; P < N; P++ ) 
    { 
   		Tmp = A[ P ];  /* the next coming card */
    	for ( j = P; j > 0 && A[ j - 1 ] > Tmp; j-- ) 
    		A[ j ] = A[ j - 1 ]; 
    	/* shift sorted cards to provide a position for the new coming card */
    	A[ j ] = Tmp;  /* place the new card at the proper position */
    }  /* end for-P-loop */
}

Shell Sort

是不稳定的排序方法，虽然比不上快排，但是代码简单。
思想是一次消除多个逆序，改进版的Insertion Sort。Define an increment sequence h1<h2<…<ht(h1=1)h_1 < h_2 < … < h_t ( h_1 = 1 )h1​<h2​<…<ht​(h1​=1)，Define an hkh_khk​-sort at each phase for k = t, t-1, …, 1。
通常我们取ht=⌊N/2⌋,hk=⌊hk+1⌋h_t=\lfloor N/2 \rfloor, h_k=\lfloor h_{k+1} \rfloorht​=⌊N/2⌋,hk​=⌊hk+1​⌋（被称为希尔增量）但是这不是最高效的增量序列，最差的情况会退化成O(N2)O(N^2)O(N2)。
取hk=2k−1h_k=2^k-1hk​=2k−1（被称为Hibbard增量）会稍微好一点，最差的情况时间复杂度为O(N32)O(N^{\frac{3}{2}})O(N23​)，平均时间为O(N76)O(N^{\frac{7}{6}})O(N67​)。

void Shellsort( ElementType A[ ], int N ) 
{ 
    int  i, j, Increment; 
    ElementType  Tmp; 
    for ( Increment = N / 2; Increment > 0; Increment /= 2 )  
        /*h sequence */
        for ( i = Increment; i < N; i++ ) { /* insertion sort */
            Tmp = A[ i ]; 
            for ( j = i; j >= Increment; j - = Increment ) 
                if( Tmp < A[ j - Increment ] ) 
                    A[ j ] = A[ j - Increment ]; 
                else 
                    break; 
            A[ j ] = Tmp; 
        } /* end for-I and for-Increment loops */
}

Heap Sort

首先我们考虑一种直白的方法：

  1. 把需要排序的元素建一个最小堆（时间复杂度为O(N)）。
  2. 每次把对堆进行DeleteMin操作，把Min存入另一个数组H。
  3. 我们获得了一个排好序的数组H。
Algorithm 1:
{
    BuildHeap( H );	/* O(N) */
    for ( i=0; i<N; i++ ) 
        TmpH[ i ] = DeleteMin( H );		/* O(lg(N)) */
    for ( i=0; i<N; i++ ) 
        H[ i ] = TmpH[ i ];		/* O(1) */
}

这是改进版的Heap Sort，可以省了额外创建一个数组的空间：

  1. 仍然是把需要排序的元素建一个最小堆。
  2. 注意到其实我们每次DeleteMin都会让堆的规模减一，所以我们把DeleteMin变成和最后一个元素进行交换，然后Percolate Down。
id Heapsort( ElementType A[ ], int N ) 
{  
	int i; 
    for ( i = N / 2; i >= 0; i-- ) /* BuildHeap */ 
        PercDown( A, i, N ); 
    for ( i = N - 1; i > 0; i-- ) { 
        Swap( &A[ 0 ], &A[ i ] ); /* DeleteMax */ 
        PercDown( A, 0, i ); 
}

Merge Sort

二分法。对于internal sorting很少使用，但是对external sorting经常使用。
时间复杂度是O(NlogN)，空间复杂度是O(NlogN)，可以是稳定的。

传统的Merge Sort算法主要是要申请额外的空间这个部分Overhead很大，下面的例程就是为了避免这个问题。

void merge_pass( ElementType list[], ElementType sorted[], int N, const int length )
{
    int left=0,mid=length-1,right=2*length-1;	
    int i,j,k;
    if (right>=N) right=N-1;
    while (mid<N)
    {
        i=j=k=0;
        //Merge [left,mid] and [mid+1,right]
        while (i<length && j<right-mid)
            if (list[i+left]<list[j+mid+1]) sorted[left+k++] = list[i++ +left];
            else sorted[left+k++] = list[j++ +mid+1];
        while (i<length) sorted[left+k++] = list[i++ +left];
        while (j<right-mid) sorted[left+k++] = list[j++ +mid+1];

        left+=2*length;
        mid+=2*length;
        right+=2*length;
        if (right>=N) right=N-1;     
    }
}

void  merge_sort( ElementType list[],  int N )
{
    ElementType extra[MAXN];  /* the extra space required */
    int  length = 1;  /* current length of sublist being merged */
    while( length < N ) { 
        merge_pass( list, extra, N, length ); /* merge list into extra */
        length *= 2;
        merge_pass( extra, list, N, length ); /* merge extra back to list */
        length *= 2;
    }
}

Quick Sort

鼎鼎有名的快速排序。最坏情况的时间复杂度是O(N2)O(N^2)O(N2)，平均情况是O(NlogN)O(NlogN)O(NlogN)。是一种不稳定的算法。

void Quicksort ( ElementType A[ ], int N )
{
    if ( N < 2 )  return;
    pivot = pick any element in A[ ]; 
    Partition S = { A[ ] \ pivot } into two disjoint sets:
    	A1={ a BelongTo S | a <= pivot } and A2={ a BelongTo S | a >= pivot };
    A = Quicksort ( A1, N1) Union { pivot } Union Quicksort ( A2, N2);
}

具体的实现（从小到大排序）

void  Qsort( ElementType A[ ], int Left, int Right ) 
{   
    int  i,  j; 
	ElementType  Pivot; 
    if ( Left + Cutoff <= Right ) {  /* if the sequence is not too short */
        Pivot = Median3( A, Left, Right );  /* select pivot */
        /* why not set Left+1 and Right-2? Because after we ++i and --j */
        i = Left;     j = Right–1;  
        for( ; ; ) { 
            /* scan from left ,when loop ends, A[i]>=Pivot */
            while ( A[++i] < Pivot );
            /* scan from right */
            while ( A[––j] > Pivot ); 
            if ( i < j ) 
                Swap( &A[ i ], &A[ j ] );  /* adjust partition */
            else     
                break;  /* partition done */
        } 
        Swap( &A[ i ], &A[ Right - 1 ] ); /* restore pivot */ 
        Qsort( A, Left, i - 1 );      /* recursively sort left part */
        Qsort( A, i + 1, Right );   /* recursively sort right part */
    }  /* end if - the sequence is long */
    else /* do an insertion sort on the short subarray */ 
        InsertionSort( A + Left, Right - Left + 1 );
}

运用快排的思想，可以很好的解决找第k大元素的方法。我们每一进行qsort，可以找出第m大的元素，如果m==k，那么就正好找到了；如果m<k继续找后面一部分；如果m>k继续找前面一部分。

Sorting Large

交换大的struct的代价可能很高。我们可以通过间接交换的方法，用一个指针指向我们真正需要排序的元素，然后对指针进行排序。等到排完了在把原来的数组排好。指针也可以是数组下标啦。

Bucket Sort and Radix Sort

桶排序就是把要排序的元素按照范围分成一个个区间（一趟扫描O(N)O(N)O(N)），然后在对区间内的元素进行排序。如果元素是均匀分布的话效果比较好。

基数排序有两种：一种是最低位优先(Least Significant Digit first)法，简称LSD法，另一种是最高位优先(Most Significant Digit first)法，简称MSD法。下面举例（下面是十进制，我觉得2^n进制更好可以除法变移位）说明：

11, 58, 97 ,81, 34, 99, 7, 32 是我们要排序的序列，最多是两位数。

  • LSD：

    最低位 桶里的内容
    0 /
    1 11, 81
    2 32
    3 /
    4 34
    5 /
    6 /
    7 97, 7
    8 28
    9 99

    然后我们得到 11,81,32,34,97,7,28,99这个序列，在对他的高位进行桶排序，直到最高位为止，就可以得到7,11,28,32,34,81,99这个正确排序的序列了。

  • MSD：

    最高位 桶里的内容
    0 7
    1 11
    2 /
    3 34,32
    4 /
    5 58
    6 /
    7 /
    8 81
    9 97

    然后我们要对子桶继续进行桶排序，直到最低位为止。我们就可以得到7,11,28,32,34,81,99这个正确排序的序列了。

这样看来，还是LSD比较方便，不用分类。但是这两个算法在实际运用中哪一个更高效还是取决于数据的情况。
