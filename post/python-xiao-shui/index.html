Python-胶水

我们知道Python被称为最好的胶水语言，利用它可以很好的粘合不同编程语言写的程序，那么具体是怎么样呢？

调用C库——ctypes

ctypes是标准库。

在我们开始调用函数前，我们必须先了解作为函数参数的 ctypes 数据类型。ctypes 是 Python 的外部函数库。它提供了与 C 兼容的数据类型，并允许调用 DLL 或共享库中的函数。可使用该模块以纯 Python 形式对这些库进行封装。

ctypes 类型 C 类型 Python 数据类型
c_bool _Bool bool (1)
c_char char 单字符字节串对象
c_wchar wchar_t 单字符字符串
c_byte char int
c_ubyte unsigned char int
c_short short int
c_ushort unsigned short int
c_int int int
c_uint unsigned int int
c_long long int
c_ulong unsigned long int
c_longlong __int64 或 long long int
c_ulonglong unsigned __int64 或 unsigned long long int
c_size_t size_t int
c_ssize_t ssize_t 或 Py_ssize_t int

c_char_p char * (NUL terminated) 字节串对象或 None
c_wchar_p wchar_t * (NUL terminated) 字符串或 None
c_void_p void * int 或 None

除了整数、字符串以及字节串之外，所有的 Python 类型都必须使用它们对应的 ctypes 类型包装，才能够被正确地转换为所需的C语言类型。

使用方法

库中的函数使用标准 C 调用约定，并假定返回 int 。通过CDLL方法调用库，返回一个库的实例：

import ctypes
libc = ctypes.CDLL("libc.so.6")
libc.printf(b"hello world")
libc.printf(b"pi %f",ctypes.c_double(3.14))

参数：可以通过设置 argtypes 属性的方法指定从 DLL 中导出函数的必选参数类型。指定数据类型可以防止不合理的参数传递（就像 C 函数的原型），并且会自动尝试将参数转换为需要的类型（printf并不是一个好的例子，他是可变参数的）：

>>> libc.printf.argtypes = [c_char_p, c_char_p, c_int, c_double]
>>> libc.printf(b"String '%s', Int %d, Double %f\n", b"Hi", 10, 2.2)
String 'Hi', Int 10, Double 2.200000
37

返回值：默认情况下都会假定函数返回 C int 类型。 其他返回类型可以通过设置函数对象的 restype 属性来指定。

libc.funcname.restype = c_char_p

指针：有时函数传参可能需要指针，byref()方法可以做到这一点，他有点像&。pointer这个运算符更加强大，是完全的&（当然也代价更大）。

from ctypes import *

libc = CDLL("libc.so.6")
i = c_int()
libc.scanf(b"%d",byref(i))
print(i.value)

ptr_i = pointer(i)
libc.scanf(b"%d",ptr_i)
print(ptr_i.contents,ptr_i.contents.value)
//你甚至可以像C一样对指针进行[]运算，不过python对他的重载好像直接取了value
print(ptr_i[0])

结构和联合：在python中也有对应的方法。我们需要继承Structure类型，然后定义_fields_字段，他是一个由2-tuple构成的list，包括字段名和字段类型。 或者你可以用3-tuple，那么最后一个元素将指定对齐方式（默认情况和C对齐方式一样）。

class POINT(Structure):
     _fields_ = [("x", c_double),
     			("y", c_double)]
"""相当于这个效果
struct POINT {
    double x;
    double y;
};
"""
point = POINT(11.5,20) 
print(point.x,point.y)

数组：我们可以用n*type的方式定义一个数组类型（class），然后实例化他，就得到了一个数组。

ARRAY = 10 * c_double
array = ARRAY(1,2,3,4,5,6,7,8,9)
scanf("%d",array)
print(ARRAY,array,array[1])

函数指针：，定义了一个函数指针cmp_func：

CMPFUNC = CFUNCTYPE(c_int, POINTER(c_int), POINTER(c_int))
def py_cmp_func(a, b):
    print("py_cmp_func", a[0], b[0])
    return 0
cmp_func = CMPFUNC(py_cmp_func)

IntArray5 = c_int * 5
ia = IntArray5(5, 1, 7, 33, 99)		//数组
qsort = libc.qsort
qsort.restype = None

qsort(ia, len(ia), sizeof(c_int), py_cmp_func)	
for i in ia: print(i, end=" ")

调用C/C++库——Cython

官方文档

Cython不是Python的标准库。[Cython]是一种编程语言，它使编写Python语言的C扩展与Python本身一样容易。它旨在成为[Python]语言的超集，从而为它提供高级，面向对象，功能和动态编程。这些功能的主要特点是支持作为语言一部分的可选静态类型声明。将源代码转换为优化的C / C ++代码，并编译为Python扩展模块。这样既可以非常快速地执行程序，又可以与外部C库紧密集成，同时又可以保持众所周知的Python语言的高程序员生产率。

安装：pip install Cython即可，不过前提是需要安装好C编译器如gcc。这在Linux上当然不是问题。

Cython是编译的魔法。

hello world

这是一个简单的python代码hello.pyx：

def say_hello_to(name):
    print("Hello %s!" % name)

这是setup.py，跟Makefile有点像，他指导python如何编译hello world.

from setuptools import setup
from Cython.Build import cythonize

setup(
    name='Hello world app',
    ext_modules=cythonize("hello.pyx"),
    zip_safe=False,
)

然后我们需要python setup.py build_ext --inplace，我们可以观察到目录里多了一个.c文件和.so文件以及一个build目录，这个.c文件就是编译的魔法生成的，这个.so文件我们就能像调用一般的库一样去调用它。

import hello
hello.say_hello_to("1")

或者是这样，加上第一行，我们就不用预先生存动态链接库.so文件了（其实第一行的任务就是帮我们生成）

import pyximport; pyximport.install()
import hello
hello.say_hello_to("1")

当然，没有这么简单，这个.pyx文件的编写语法是一个混合了C和Python风格的可以说是新的语言的产物。我们要通过这些语法来指导编译器生成更好的代码。我们可以在这个.pyx代码里调用C库，定义静态变量、使用指针等等C操作，甚至可以有C++的类，也可以使用普通python的特性。然后魔法会把他编译好。

调用JAVA——jpype

官网

JPype致力于允许python程序完全访问java类库。这不是通过像Jython / JPython一样通过重新实现Python来实现的，而是通过在两个虚拟机中的本机级接口实现的。

最终，在许多（尽管不是全部）情况下，应该有可能用python替换Java。JSP，Servlet，RMI服务器和IDE插件是不错的选择。

一旦实现了这种集成，便将开始第二阶段，将Java逻辑与Python逻辑分离，最终允许将该桥接技术用于其他环境，例如IE Ruby，Perl，COM等。
