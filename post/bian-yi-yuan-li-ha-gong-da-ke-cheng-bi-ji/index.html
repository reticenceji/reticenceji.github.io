语法分析

一个**上下文无关文法（context free grammar，简称文法）**由下面四个元素组成：

  1. 一个终结符号(terminal symbol)集合，有时也叫词法单元(token)。
  2. 一个**非终结符号(nonterminal symbol)**集合，又是也叫语法变量。每个非终结符号表示一个终结符号串的集合。
  3. 一个**产生式(production)**集合
  4. 指定一个非终结符号位开始符号(start symbol)。

例如：stmt -> if (expr) stmt else stmt这样的规则叫做产生式，if和()被称为终结符号，expr和stmt这样的变量表示终结符号的序列，称为非终结符号。可以利用文法进行推导和归约。

（上下文无关文法的）推导：从开始符号出发，不断将某个非终结符号替换成该终结符号的某个产生式的体。可以从开始符号推导得到的所有终结符号串的集合称为改文法定义的语言。

其实一共有四种文法，逐级限制 α→β：

  • 0型文法：α中至少包含1个非终结符
  • 1型文法（CSG 上下文有关） ：｜α｜≤｜β｜
  • 2型文法（CFG 上下文无关文法） ：α ∈ 非终结符
  • 3型文法（RG 正则）：A→wB 或 A→w (A→Bw 或A→w)

上下文无关文法的分析树(parse tree)。其实是推导的图形化表示：

  1. 分析树的根节点为文法的开始符号。
  2. 内部节点表示对一个产生式α→β的应用。
  3. 叶节点可以是终结符也可以是非终结符，从左到右排列叶节点得到的符号串称为是这棵树的产出(yield)或者边缘(frontier)。

自顶向下的分析方法

自顶向下的分析方法：从分析树的顶部（根节点）向底部（叶节点）方向构造分析树。有两个问题：

  1. 替换当前句型中的哪个非终结符：最左推导，总是选择每个句型中最左非终结符进行替换（自顶向下采用）；最右推导，总是选择每个句型中最右非终结符进行替换。
  2. 用该终结符的哪个候选式进行替换。根据输入流中的下一个终结符，选择最左非终结符的一个候选式，这个可能会有多个，就需要枚举并回溯，这会影响效率。 为此，我们有预测分析法，预测分析法的工作过程，从文法开始符号出发，在每一步推导过程中根据当前句型的最左非终结符A和当前输入符号a，选择正确的A-产生式。为保证分析的确定性，选出的候选式必须是唯一的。预测分析不需要回溯，是一种确定的自顶向下分析方法。通过向前看k个符号来选择正确的产生式，称为LL(k)文法。
从这里开始下面的内容介绍的都是预测分析法

并不是所有的文法都适合预测分析。比如：

  1. 同一非终结符的多个候选式存在共同前缀，将导致回溯现象。解决算法如下：

    // 提取左公因子算法
对于每个非终结符A，找出它的两个或多个选项之间的最长公共前缀α。如果α ≠ ε，
即存在一个非平凡的( nontrivial )公共前缀，那么将所有A-产生式
    A → α β1 | α β2 | … | α βn | γ1 | γ2 | … | γm
替换为
    A → α A' | γ1 | γ2 | … | γm
    A' → β1 | β2 | … | βn
其中， γi 表示所有不以α开头的产生式体； A'是一个新的非终结符。不断应用这个转换，直到每个非终结符的任意两个产生式体都没有公共前缀为止。

  2. 左递归文法会使递归下降分析器陷入无限循环。 具体的消除方式如下：

// 消除左递归算法
// 输入：不含循环推导（即形如A =>+ A的推导）和 ε-产生式的文法G
// 输出：等价的无左递归文法

按照某个顺序将非终结符号排序为A1，A2，… ，An .
for ( 从1到n的每个i ) {
    for ( 从1到i -1的每个i ) {
        将每个形如Ai → Aj γ的产生式替换为产生式组 Ai -> δ_1γ ∣ δ_2γ ∣ ... ∣ δ_k γ ，
        其中Aj -> δ_1∣δ_2∣ ... ∣δ_k ，是所有的Aj 产生式
     }
     消除Ai 产生式之间的立即左递归
}

LL1文法

符号约定和回顾：

  1. ->+ 通过>0步推导可以得到。->*通过>=0步推导可以得到。

  2. FOLLOW：可能在某个句型中紧跟在A后边的终结符a的集合。

  3. FIRST：给定一个文法符号串α， α的串首终结符集FIRST(α)被定义为可以从α推导出的所有串首终结符构成的集合。如果α -> * ε，那么ε也在FIRST(α)中。

  4. SELECT：产生式A->a的可选集是指可以选用该产生式进行推导时对应的输入符号的集合，记为SELECT( A->β )，有以下规则

    如果ϵ∉FIRST(a)\epsilon \notin FIRST(a)ϵ∈/​FIRST(a)​，SELECT(A->a) = FIRST(a)

    如果ϵ∈FIRST(a)\epsilon \in FIRST(a)ϵ∈FIRST(a)​，SELECT(A->a) = FIRST(a) - {ε} U FOLLOW(A)

S_文法（简单的确定性文法，显然，满足这个文法是可以很简单的预测分析的）：

  1. 每个产生式的右部都以终结符开始
  2. 同一非终结符的各个候选式的首终结符都不同

q_文法

  1. 每个产生式的右部或为ε ，或以终结符开始
  2. 具有相同左部的产生式有不相交的可选集(SELECT)

LL1 文法，第一个“L”表示从左向右扫描输入，第二个“ L”表示产生最左推导，“1”表示在每一步中只需要向前看一个输入符号来决定语法分析动作。

计算

FIRST集的计算。结合FIRST集的定义，其实挺直观的。FIRST ( X )：可以从X推导出的所有串首终结符构成的集合

FOLLOW集的计算。FOLLOW(A)：可能在某个句型中紧跟在A后边的终结符a的集合。如果A是某个句型的最右符号，则将结束符$添加到FOLLOW(A)中。

SELECT集的计算。有了前两个的基础可以来计算SELECT集了。我们只需要应用上面的两个规则即可。产生式A->a的可选集是指可以选用该产生式进行推导时对应的输入符号的集合，记为SELECT( A->β )。对于LL1文法，具有相同左部的产生式的SELECT集互不相交。

  • 如果ϵ∉FIRST(a)\epsilon \notin FIRST(a)ϵ∈/​FIRST(a)，SELECT(A->a) = FIRST(a)
  • 如果ϵ∈FIRST(a)\epsilon \in FIRST(a)ϵ∈FIRST(a)​​​，SELECT(A->a) = FIRST(a) - {ε} U FOLLOW(A)

从这里开始主要介绍移入-归约分析法

自底向上的分析方法

  • 移入：将下一个输入符号移到栈的顶端
  • 归约：被归约的符号串的右端必然处于栈顶。语法分析器在栈中确定这个串的左端，并决定用哪个非终结符来替换这个串
  • 接收：宣布语法分析过程成功完成
  • 报错：发现一个语法错误，并调用错误恢复子例程

回顾一个重要知识点：

令G是一个文法，S是文法G的开始符号。假设αβθ\alpha\beta\thetaαβθ是文法G的一个句型，如果有

S→∗αAθ,A→∗βS \rightarrow^* \alpha A\theta , A\rightarrow^*\beta S→∗αAθ,A→∗β

则称β\betaβ是句型αβθ\alpha\beta\thetaαβθ相对于非终结符A的一个短语。从语法树上看，一个句型的语法树中任一子树的叶节点所组成的符号串都是该句型的短语。

特别的，如果有A→βA\rightarrow\betaA→β​则称β\betaβ​是句型αβθ\alpha\beta\thetaαβθ​相对于非终结符A的一个直接短语。一个句型的语法树中任一**最小子树(两层)**的叶节点所组成的符号串都是该句型的短语。

句柄，是最左边的直接短语。

LR分析的关键是识别句柄。

LR文法

LR文法是最大的，可以构造出相应的移入-归约语法分析器的文法类。L: 对输入进行从左到右的扫描；R: 反向构造出一个最右推导序列。LR(k)分析：需要向前查看k个输入符号的LR分析。k = 0 和 k = 1 这两种情况具有实践意义。当省略(k)时，表示k =1。

关键是分析表的构造。动作表有两种动作

  • sn：将符号a、状态n 压入栈
  • rn：用第n个产生式进行归约

文法中的项目 :

右部某位置标有圆点的产生式称为相应文法的一个LR(0)项目（简称为项目）。项目描述了句柄识别的状态。例如，S->bBB对应的项目：

S->·bBB    移进项目
S->b·BB    \待约
S->bB·B    /项目
S->bBB·    归约项目

可以把等价的项目组成一个项目集( I ) ，称为项目集闭包(Closure of Item Sets)，每个项目集闭包对应着自动机的一个状态。等价简单的说，就是如果有项目A→α⋅BβA \rightarrow \alpha · B\betaA→α⋅Bβ​，并且有B→⋅γB\rightarrow · \gammaB→⋅γ​​，（这个·可以理解做等待，如果在等待B，那么不就是在等待B后面的东西么）那么这两个项目是等价的，等价是自反、传递、互逆的。有了等价状态就可以构造自动机了。

LR(0)

算法：计算状态（项目集闭包）

SetOfltems CLOSURE ( I ) {
    J = I;
    repeat
        for ( J中的每个项A → α·Bβ ) 
            for ( G的每个产生式B → γ ) 
                if ( 项B → · γ不在J中 ) 
                将B → · γ加入J中;
    until 在某一轮中没有新的项被加入到J中;
    return J； 
}

计算GOTO（返回项目集I对应于文法符号X的后继项目集闭包）

SetOfltems GOTO ( I，X ) {
    将J 初始化为空集;
    for ( I 中的每个项A → α·Xβ ) 
        将项 A → αX·β 加入到集合J 中;
    return CLOSURE ( J );
}

构造LR(0)自动机的状态集

void items( G' ) {
    C＝{ CLOSURE ({[ S'→ ·S ] } ) }；
    repeat
        for (C中的每个项集 I )
            for(每个文法符号X )
                if ( GOTO ( I，X )非空且不在C中) 将GOTO ( I，X )加入C中；
    until在某一轮中没有新的项集被加入到C中；
}

对于LR(0)文法，以上处理方式不会产生冲突。准确来说，以上处理方式不会产生冲突的我们称为LR(0)文法。

词法分析

正则表达式

这个是老朋友了。正则文法。C语言中标识符的正则定义

  • digit → 0|1|2|…|9
  • letter_ → A|B|…|Z|a|b|…|z|
  • id → letter(letter_|digit) 【变量名】
  • digit → 0|1|2|…|9
  • digits → digit digit
  • optionalFraction → .digits|ε
  • optionalExponent → ( E(+|-|ε)digits )|ε
  • number → digits optionalFraction optionalExponent 【字面量数】

有穷自动机

有穷自动机：确定的自动机，DFA；不确定的自动机，NFA。区别可以看0，NFA当状态为0输入为a的时候可以有不止一个达到的状态，NFA更直观，DFA更适合计算机实现（感觉就是状态机？）。

如果用正则表达式的话，就是r=(a|b)*(abb)。

如果用正则文法表示，就是

B -> Aabb | abb
A -> Aa | Ab | a | b

一般来说，我们采用这样的方式： RE->NFA->DFA->program language

语法制导翻译

语法制导定义SDD : 将每个文法符号和一个语义属性集合相关联，将每个产生式和一组语义规则相关联，这些规则用
于计算该产生式中各文法符号的属性值。

综合属性：分析树节点N上的 非终结符A的继承属性只能通过 N的子节点和N本身 来决定。

继承属性：分析树节点N上的 非终结符A的继承属性只能通过 N的父节点、兄弟节点和N本身 来决定。

S-SDD

仅使用综合属性的SDD称为S-SDD，如果一个SDD是S属性的，可以按照语法分析树节点的任何自底向上顺序来计算它的各个属性值。S-属性定义可以在自底向上的语法分析过程中实现。

L-SDD

直观含义：在一个产生式所关联的各属性之间，依赖图的边可以从左到右，但不能从右到左(因此称为L属性的，L是Left的首字母)。是S-SDD的扩展。

语法制导翻译方案SDT

语法制导翻译方案(SDT )是在产生式右部中嵌入了程序片段(称为语义动作)的CFG。

如果一个S-SDD的基本文法可以使用LR分析技术，那么它的SDT可以在LR语法分析过程中实现。

语义动作中的抽象定义式改写成具体可执行的栈操作，在遇到归约操作的时候，应用SDT。
