GCD相关定理

设a、b为整数，且a、b中至少有一个不等于0，令d=gcd(a,b)d=gcd(a,b)d=gcd(a,b)，则一定存在整数x、y使得下式成立：a∗x+b∗y=da*x + b*y = da∗x+b∗y=d。
特别地，当a、b互素时，则一定存在整数x、y使得a∗x+b∗y=1a*x+b*y=1a∗x+b∗y=1成立。
计算乘法逆元：

a*b = c mod n
a*b = 1 mod n
比如下面这个例子，我用5作为加密密钥，把4加密成6。解密使用5的“倒数”3。
这样让明文和密文落在同一个区间内，且不需要除法。
4 * 5 = 6 mod 7
6 * 3 = 4 mod 7

逆元

加法逆元 a+b=0a+b=0 % na+b=0
乘法逆元 a∗b=1a*b=1 % na∗b=1
防止出现减法和除法，这样就不会出现非自然数了。(他们不是群的运算符)。
一个经典的应用就是凯撒加密算法。

求加法逆元有手就行。求mod 35下13的乘法逆元x。就是求x∗13≡1mod 35x * 13 \equiv 1 mod\ 35x∗13≡1mod 35。
上面的定理告诉我们，gcd(13,35)=1

ECB和CBC和CFB（Cipher Block Chaining）
ECB模式的弱点是:对于相同内容的明文段，加密后得到的密文块是相同的。ECB模式的优点是:加密和解密过程均可以并行处理。
CBC模式的特点是：当前块的密文与前一块的密文有关; 加密过程只能串行处理; 解密过程可以并行处理。
CFB模式的优点是可以从密文传输的错误中恢复。

DES 对称密钥加密算法⭐️

https://www.ruanx.net/des/
明文是64位=8字节。密文也是64位=8字节。密钥是64位=8字节（或者说是56 bit，因为每个byte的第8位是没有用的）
加密与解密的密钥相同，所以是对称算法。
对DES的攻击是在已知部分明文、密文对的情况下，解出密钥。有效的攻击方法是差分分析。

DES是把64 bit明文通过56 bit密钥加密成64 bit密文的算法

DES的加密过程如下。一开始先是一个初始置换IPIPIP，把明文分解成L0,R0L_0,R_0L0​,R0​，然后送进Feistel网络。

Feistel网络一共进行十六轮操作，第i次操作可以表示成

  • Li=Ri−1L_i=R_{i-1}Li​=Ri−1​
  • Ri=Li−1⨁f(Ri−1,ki)R_i=L_{i-1} \bigoplus f(R_{i-1},k_i)Ri​=Li−1​⨁f(Ri−1​,ki​)。这个 kik_iki​是什么呢？kik_iki​是一开始的密钥k通过密钥编排来实现的。

经过十六轮操作之后，再次交换Li,RiL_i,R_iLi​,Ri​，然后通过逆初始置换IP−1IP^{-1}IP−1得到加密的密文。

    1. 首先把输入分成8个4 bit的分组，然后把每个分组扩展成6 bit，从而将32 bit扩展成48 bit。这就是下图中的E-盒。32个输入位有16个会在输出中出现两次，但是任何一个输入位都不会在每个6 bit的分组中出现两次，也不会不改变位置。Expansion的规则。
    2. 然后把得到的48 bit的结果和轮密钥kik_iki​进行XOR，得到8组6bit共48bit输出。

    4. 最后，再进行置换P。置换P将扩散引入DES。P置换的规则。

差分分析

差分分析是较有效的一种破解DES的方法。可以用来进行选择明文攻击。

2DES和中间相遇攻击

2DES：c = des(des(p,k1),k2);

用256个密钥k1加密p得到256个密文c'

对得到的密文c'进行排序（O(NlogN))

用2^56个k2对c进行解密得到p'(O(N))

​ 我们在{c'}中查找p'(O(logN))，如果找到p'==c'就解密成功。

所以总共的复杂度是O(NlogN)，不过这个N有2^56。

2DES的安全性不好，所以我们用3DES。

3DES：c = E(D(E(p,k1),k2),k3)

Definition Group
A group is a set of elements G together with an operation ◦ which combines two elements of G. A group has the following properties:

  1. The group operation ◦ is closed. That is, for all a,b,∈ G, it holds that a ◦ b = c ∈ G.
  2. The group operation is associative. That is, a◦(b◦c)=(a◦b)◦c for all a,b,c ∈ G.
  3. There is an element 1 ∈ G, called the neutral element (or identity element), such that a ◦ 1 = 1 ◦ a = a for all a ∈ G.
  4. For each a ∈ G there exists an element a−1 ∈ G, called the inverse of a, such that a ◦ a−1 = a−1 ◦ a = 1.
  5. A group G is abelian (or commutative) if, furthermore, a ◦ b = b ◦ a for all a,b ∈ G（交换群）

Definition 4.3.2 Field
A field F is a set of elements with the following properties:

  1. All elements of F form an additive group with the group operation “+” and the neutral element 0.
  2. All elements of F except 0 form a multiplicative group with the group operation “×” and the neutral element 1.
  3. When the two group operations are mixed, the distributivity law holds, i.e., for all a,b,c ∈ F: a(b+c)=(ab)+(ac).

哈希算法

严格的说，哈希算法不是加密算法，因为是单向不可逆的。但是和数字签名有很大的关系。

什么是签名呢？假设有一封邮件L。
m=md5(L), m'=RSA(m, 私钥)
然后把L+m'发送给接受者
M = RSA(m',公钥)，然后比较M==md5(L)
若相等，则邮件L可靠。

MD5

MD5源代码
MD5是以64byte为单位进行的。他会把数据利用填充政策，分成64 bytes的块。
然后对块进行计算，计算结果依赖于前一个块的计算结果。
编译的时候要链接上库文件。

#include <openssl/md5.h>
#include <string.h>
int main{
    MD5(s,strlen(s),t);     //最简单的调用md5的办法
    //正规的调用办法
    char o[1000]={0};
    MD5_CTX t;
    MD5_Init(&t);
    MD5_Update(&t,"123",3);
    MD5_Update(&t,"456",3);
    MD5_Final(o,&t);
}
