一个进程是怎么开始的

2020-11-21 5 min read # Reverse # Linux

Perface

最开始，我们可能会认为程序是从main开始执行的。
了解深入之后，以Linux为例，一个进程的开始，往往是先fork，然后exec。在这个过程中，寄存器什么的环境被设置，具体发生了什么呢？
我们还知道一个ELF文件里面有entry point表示程序的入口地址。通过观察一个程序的反汇编代码，我们知道他指向的并不是main，之前还有别的准备工作。那是什么呢。
带着这些疑问，开始探索。（未完待续）

这里的操作都在干什么呢？其实都是在准备__libc_start_main的参数然后call __libc_start_main。至于具体的参数准备，结合下面介绍吧。
注意一下在我试验的平台（64位Linux）的函数调用规范是：第1到第6个参数rdi，rsi，rdx，rcx，r8，r9，超出6个的参数从右向左放入栈中。
以及当_start开始时的堆栈情况：

__libc_start_main

我们来看他的函数原型。

STATIC int LIBC_START_MAIN (
    int (*main) (int, char **, char **),
    int argc,
    char **argv,
    __typeof (main) init,
    void (*fini) (void),
    void (*rtld_fini) (void),
    void *stack_end
)

他是做什么用的呢？call the main function with appropriate arguments, and handle the return from main().

_init和_fini

分别是constructor和destructor。什么意思呢？他们分别是调用动态链接库时的初始化函数和释放函数。具体是怎么工作的我也不是很清楚。

我们通过readelf可以查看到有这两个名字的段

  [14] .fini             PROGBITS         00000000000006e4  000006e4
