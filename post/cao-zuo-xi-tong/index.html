2. Process

2.1. Process Concept

Process = code section+ program counter + register + stack + data section + heap
进程，是组织资源的最小单元。可以简单的认为process 是 program in execution.
在/proc/pid中查看进程信息。Memory的layout可以查看maps文件。如下图所示

可以看出，上面是程序部分。text section存存放代码，rodata只读数据，data存放数据。
下面是heap，顶上是stack，中间是用到的链接库。
当stack增长到lib的代码的时候，就会stackoverflow。

2.2. Process State：

这些状态具体随操作系统的不同而不同。

  • New：fork()创建一个进程。
  • Running
  • Waiting
  • Ready
  • Terminate
    在Linux中，一个process的parent process如果terminate了，那它就是orphan进程。会被pid=1的进程收养。
    如果process terminate了，但是没有被父进程回收，就变成了僵尸进程（zombie process）

2.3. Process Control Block

PCB保存很多Process的重要信息，每个进程被一个PCB代表。(好吧，不是这样的。应该说每个线程都有一个PCB，进程是一个虚拟的概念)
PCB有如process state, program counter, CPU reg, CPU-scheduling information, Memory-management information, accounting information, IO status information.

PCB：在Linux中是task_struct结构

  • 源代码：https://elixir.bootlin.com/linux/latest/source/include/linux/sched.h#L629
  • 存储着重要的关于进程的信息
  • waiting、ready队列会存储PCB以备running

2.4. Thread

线程是最小的执行单元。一个进程可以有多个线程，这样在多核CPU中尤其高效。并且context switch也更快。

一个Thread特有：thread ID,PC,Stack,Register set：这些构成了thread block。TCB ，在Linux中也用task_struct结构描述。LWP（light weight process）在Linux中就指的是Thread。
但是共享：code section,data section,heap,opend files；也就是说，thread之间的通信比IPC更加高效，但是也更危险。
现代的操作系统中每个process至少有一个thread。我们可以看到one Thread的Thread ID就是Process ID，mutiple Thread的ID
