如何登录

我们需要实现我们自己的User Class。
官方的要求如上。但是就我自己实现来看，首先继承UserMixin，然后我们只需要让我们的类中有id这个属性就好。
其次，由于我们还要实现load_user这个函数，所以我下面又写了一个get方法。这可能不是最佳实现，但是够用。

class User(UserMixin):
    users = []

    def __init__(self,row):
        self.id = row['id']
        self.password = row['password']
        self.name = row['name']
        self.phone = row['phone']
        User.users.append(self)

    @classmethod
    def get(cls,id):
        for u in cls.users:
            if u.id == id:
                return u
        return None

@login_manager.user_loader
def load_user(userid):
    return User.get(userid)

然后的话，我们的login和logout就只需要简单的调用函数login_user, logout_user就可以了：

@app.route('/login',methods=['GET', 'POST'])
def login():
    form = forms.LoginForm()
    if form.validate_on_submit():
        admintable = all_table['admin']
        user = db.session.query(admintable).filter_by(id=form.id.data).first()
        if user is not None:
            user = User(user._mapping)
        if user.password == md5(form.password.data.encode('ascii')).digest():
            login_user(user)
            return redirect(url_for('index'))
        flash('用户名或密码不正确')
    
    return render_template('login.html', form=form)

@app.route('/logout')
@login_required
def logout():
    logout_user()
    flash('你已经退出')
    return redirect(url_for('index'))

flask-wtf中的表单类型

创建一个表单类的基本方法：
继承的基类以前叫Form，新版本改名叫FlaskForm了

from flask_wtf import FlaskForm
from wtforms import BooleanField,StringField,IntegerField,DecimalField,SubmitField\
    ,RadioField,DateField,SelectField,PasswordField
from wtforms.validators import Required,AnyOf,Optional

class BookAddForm(FlaskForm):
    isbn = StringField('ISBN',validators=[Required()])
    classify = SelectField('分类',choices = ['A','B','C','D','E'],validators=[Required(),AnyOf(['A','B','C','D','E'])])
    name = StringField('书名',validators=[Required()])
    number = IntegerField('数量',validators=[Required()])
    
    press = StringField('出版社')
    year = StringField('出版日期 Y-M-D',validators=[Optional()])
    author = DateField('作者')
    price = DecimalField('价格')
    
    submit = SubmitField("添加")

有一个基础的表单使用总结：

  1. 编写forms.py文件， 定义一个关于表单的类;(***注意： 一定要有提交的按钮);

  2. 业务逻辑文件app.py中，

    • 实例化表单类; form = RegisterForm()
    • 验证表单内容是否提交成功? form.validate_on_submit():
    • 获取表单里面的内容(两种方法):
      • 通过request对象获取: request.form['key值']
      • 通过form对象获取: form.key.data

  3. 表现逻辑文件wtf.html中:

    • 导入wtf模块: {% import "bootstrap/wtf.html" as wtf %}
    • 自动生成表单对应的html： {{ wtf.quick_form(form) }}

同时两个选择器给同一个元素设置相同的属性，那么最终设置为哪个呢？这就要按照优先级来了

  • 内连样式 优先级 1000
  • id 选择器 优先级 100
  • 类选择器 优先级 10
  • 元素选择器 优先级 1
  • 统配选择器 优先级 0
  • 继承的样式优先级没有

以上的优先级先满足高的。如果是两种相同优先级 为同一个元素 同一个属性设置 的话，是哪个写在代码靠后 最终就按那个的样式
