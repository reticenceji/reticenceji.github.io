GCC-内嵌汇编

2020-11-09 2 min read # C

因为C语言翻译成汇编语言并不复杂，内嵌汇编也并不复杂。但是汇编和硬件强相关，所以还是要结合具体架构。

参考链接

方法

使用方法：__asm__ __violate__ (汇编语句 : 输出 : 输入 : 破坏寄存器)，其中violate是告诉GCC不要做优化。

  • 汇编语句：

    • 在一个字符串内完成，用;分隔语句。为了美观可以用\换行。
    • 具体的语句和架构有关。如果用到了后面的输入输出部分的寄存器可以用%index指定，见demo。

  • 输出："=类型"(C语言元素)

  • 输入："类型"(C语言元素)

    • 类型主要有"r"寄存器，"m"内存，"i"立即数

  • 破坏描述："破坏的寄存器"

    • 告诉GCC哪些寄存器的内容被改变了

Demo

我们来看几个demo：

RISCV

__asm__(
        "csrw sepc,%0;\
         sret;"
        :
        :"r"(dead_loop));
)

x86，注意是AT&T格式的汇编。

    int flag, new_flag;
    __asm__
    (
    "movl %1, %%eax \n"
    "orw $2, %%ax \n"
    "movl %%ax, %0 \n"
    : "=r"(new_flag) /* output */
    : "r"(flag)      /* input */
    : "%eax"         /* clobbered register */
    );
