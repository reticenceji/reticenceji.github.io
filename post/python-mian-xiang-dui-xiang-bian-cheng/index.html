Python-面向对象编程

2020-11-11 Views Python1907字9 min read繁

参考链接：标准库，realpython教程 非常好的教程：python-course

Everything is a class in Python.，我们可以在下面看到数字、列表这些都是类。实现了那些“接口”，你也可以创造数字类型、可枚举类型。目前先看到这里，以后用到更深的技巧了再说。

类和实例基础

class and instance. 实例是类的创建出来的对象。

class Student(object):
	pass

zxy = Student()

属性和方法是类的基础。

属性 Attribute

Python的类和实例都可以有Attribute。我们可以观察到Attribute是存储在__dict__中的。
当我们试图访问一个不存在的属性，如x.nama时，会产生一个异常。

class Robot():
	def __init__(self,name):
		self.name = name

	def hi(self):
		print("hello world")

x = Robot("JGQ")
print(x.name,x.__dict__)
>>> JGQ {'name': 'JGQ'}
print(Robot.__dict__)

我们要给类定义Attribute，应该在类的层面：

class Robot():
	factory = "ZhongTai"	#这是类的属性

这样，就算你在某个实例中定义一个同名的Attribute也不会改变或覆盖类的Attribute。当然，这样在这个实例中这个Attribute就不可见了。

方法 Methods

可以认为方法是定义内Class内的Function，他和Function主要有两点不同：

  1. It belongs to a class, and it is defined within a class.（注意是属于Class，我们可以在__dict__中看到类的方法）
  2. The first parameter in the definition of a method has to be a reference to the instance, which called the method. This parameter is usually called "self".

静态方法 static method

我们用 @staticmethod修饰静态方法后，我们可以通过类或实例访问静态方法。否则我们只能通过类访问，因为如果通过实例访问的话，会传入额外的参数self。

class Robot:
	__counter = 0
	
    @staticmethod
	def RobotInstances():
		return Robot.__counter

	def __init__(self):
		type(self).__counter += 1

x = Robot()
print(Robot.RobotInstances())

类方法 class method

Like static methods class methods are not bound to instances, but unlike static methods class methods are bound to a class。他们的主要区别在于是否有一个cls的传参。（cls并不是什么关键字，但是约定俗称，指代该class，就像self指代那个实例一样。）

class fraction(object):

    def __init__(self, n, d):
        self.numerator, self.denominator = fraction.reduce(n, d)
        

    @staticmethod
    def gcd(a,b):
        while b != 0:
            a, b = b, a%b
        return a

    @classmethod
    def reduce(cls, n1, n2):
        g = cls.gcd(n1, n2)
        return (n1 // g, n2 // g)

    def __str__(self):
        return str(self.numerator)+'/'+str(self.denominator)

Q：那么Class Methods && Static Methods && Instance Methods的区别是什么呢？

A：个人理解，static methods是一个独立于类的方法，就像一个函数，没有特殊参数。Class Method是和类同一个层级的，传入cls参数表示这个Class，Instance Method是和实例一个层级的，传入self参数表示这个Instance。
如果我们的函数用到实例的Attribute应该用Instance Method，用到类的Attribute应该用Class Method。如果都没有可以用static methods。

属性的可见性

python中是没有所谓的private/public修饰符的。采用下面的方法限定属性。

Naming Type Meaning
name Public These attributes can be freely used inside or outside a class definition.
_name Protected Protected attributes should not be used outside the class definition, unless inside a subclass definition.
__name Private This kind of attribute is inaccessible and invisible. It's neither possible to read nor write to those attributes, except inside the class definition itself. 如果我们调用的话，会提示我们没有这个属性。

Getter and Setter

参考链接：https://www.python-course.eu/python3_properties.php

如果熟悉了Java可能会很喜欢定义一个Private变量，然后提供get()和set()。但是这个做法一点也不Python。

class P:
    def __init__(self,x):
        self.x = x

    @property
    def x(self):
        return self.__x

    #注意，@property定义了x这个符号。
    @x.setter
    def x(self, x):
        if x < 0:
            self.__x = 0
        elif x > 1000:
            self.__x = 1000
        else:
            self.__x = x
            
xy = P(1023)	#我们观察初始化函数，其实也调用了setter
xy.x = 123		#@x.setter的调用方法
print(xy.x)		#@property的调用方法

A method which is used for getting a value is decorated with "@property", i.e. we put this line directly in front of the header. The method which has to function as the setter is decorated with "@x.setter". 太优美了。

那么我们应该如何选择变量的定义呢？以变量X为例。

  1. 我们类的可能用户是否需要X的值？
  2. 如果没有，我们可以或者应该将其作为Private Attribute。
  3. 如果必须访问它，我们将其作为Public Attribute进行访问。
  4. 当且仅当我们必须对数据进行一些检查(@x.setter)或转换(@property)时，我们将它定义为具有相应属性的私有属性。

继承 Inheritance

class Animal(object):
    def run(self):
        print('Animal is running...')

我们可以看到Animal继承了object类。继承可以获得父类的所有方法。Python支持多重继承。

类似的，如果子类和父类有同名的方法，可以实现Overriding。子类的方法可以覆盖父类的方法。如果我们想在父类的方法上添加新特性，可以用super()调用父类的方法。

  • type(ins)：返回对象的类型
  • isinstance(ins,class)：判断一个对象是否是一个类的实例。父类也会返回True.

类的特殊方法

或者说是object类的方法。显然所有的类都会继承object类的这些方法。

  1. __init__，__new__
    这两个方法共同创建一个类的实例。对于我们来说只要掌握__init__即可。
class Student(object):
	def __init__(self,name,age):
		self.name = name
		self.age = age

zxy = Student("zxy",18)

  1. __repr__和__str__
    __repr__这个方法主要用于调试。Called by the repr() built-in function to compute the “official” string representation of an object 。
    __str__，Called by str(object) and the built-in functions format() and print() to compute the “informal” or nicely printable string representation of an object。也就是说，当我们试图print(objA)的时候，我们其实就是在print(objA.__str__())
    是不是觉得这个东西就像“接口”呢？我们接着往下看。

  2. __bytes__
    Called by bytes() to compute a byte-string representation of an object. This should return a bytes object.
    但是我试图[123].__bytes__()的时候给我报错了？

  3. __hash__

    Called by built-in function hash().主要是给对于set/dict这种需要hash的数据结构用的。对于这个方法，唯一的要求是如果两个类相等那么他们的hash值也要是一样的。所以，如果一个class没有定义__eq__方法那么他也不应该定义__hash__方法。

  4. __bool__

    Called to implement truth value testing and the built-in operation bool(); should return False or True.如果未定义此方法，则会查找并调用 __len__() 并在其返回非零值时视对象的逻辑值为真。如果一个类既未定义 __len__() 也未定义 __bool__() 则视其所有实例的逻辑值为真。
