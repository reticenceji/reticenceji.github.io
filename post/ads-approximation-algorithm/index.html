ADS-Approximation algorithm

2021-05-29 Views DS1319字7 min read繁

许多有实际意义的问题是NPC问题。如果规模小的话我们可以求精确解，但是对于规模大的问题，我们就可以用多项式复杂度的算法求近似最优解。
Approximation Ratio: ρ(n)=max(C/C∗,C∗/C)\rho(n) = max(C/C^*,C^*/C)ρ(n)=max(C/C∗,C∗/C)。

(1+ ε)-approximation algorithm: 除了问题的输入，还要输入希望求得解的精度ε,ratio是1+ε。

We say that an approximation scheme is a polynomial-time approximation scheme (PTAS) if for any fixed ϵ>0\epsilon>0ϵ>0, the scheme runs in time polynomial in the size n of its input instance. 比如O(n2/ϵ)O(n^{2/\epsilon})O(n2/ϵ)对任何ϵ\epsilonϵ都有一个多项式复杂度算法。

In PTAS algorithms, the exponent of the polynomial can increase dramatically as ε reduces, for example if the runtime is O(n(1/ε)!)O(n^{(1/ε)!})O(n(1/ε)!) which is a problem. There is a stricter scheme, Fully Polynomial Time Approximation Scheme (FPTAS). In FPTAS, algorithm need to polynomial in both the problem size n and 1/ε. 像上面那个例子就不是FPTAS，而O((1/ϵ)2n3)O((1/\epsilon)^2n^3)O((1/ϵ)2n3)就是FPTAS。

我确实听不懂这些东西，只能记住了。

The K-center Problem

vertex cover

在图论的数学学科中，图的顶点覆盖（有时是节点覆盖）是一组顶点的集合，使得图的每个边缘至少与集合中的一个顶点相连接。 他是NPC问题。

一个2-approximation算法

C = {}
E' = G.E
while E != {};
  let (u,v) be an arbitrary edge of E'
  C = C U {u,v}
  remove from E' every edge incident on either u or v
return C

证明为什么是2：

如果我们的近似算法用的顶点集合为CCC，边集合为AAA，而最优化算法用顶点C∗C^*C∗。

首先，因为近似算法实现了节点覆盖，而AAA中不存在两条边拥有同一个顶点，所以∣C∗∣≥∣A∣|C^*|\ge |A|∣C∗∣≥∣A∣。

算法第四行每次执行都会选出一条边，两个端点都不在C中，因此返回顶点覆盖的上界是∣C∣=2∣A∣|C|=2|A|∣C∣=2∣A∣。

故2∣C∗∣≥∣C∣2|C^*|\ge|C|2∣C∗∣≥∣C∣。
