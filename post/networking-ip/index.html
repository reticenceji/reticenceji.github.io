Networking-网络层-IP

2021-02-05 5 min read # Networking



IP是使用Internet的细腰，Network层只能使用IP。

IP

IP是Internet Protocol的缩写，只要想通过Internet进行连接就必须使用IP协议。IP datagrams由header和data组成。IP协议是一个简单的协议，也是最重要的协议。

Property Behavior
Datagram 传递的是互相独立的Datagram，SA DA Data
Unreliable Packet有可能丢失
Best effort 尽最大的努力完成Packet的传递，但是不做任何保证，不会检测错误
Connectionless 无连接，不保证Packet的顺序

End-to-end原则：只要能在端点实现的功能，就在端点实现。所以我们不让IP成为一个可靠的服务，而是让TCP成为可靠的服务，当然这是原因之一。

IP提供的服务

  1. 防止packet死循环。
  2. 将把太长的Packet分段（比如，下一层只能一次传输1500byte，但是一个Packet超过了这个大小）。这也意味着我们需要把它正确的按顺序重组。
  3. 我们有对Header进行检验的Checksum，减少数据包传递到错误地方的可能性。
  4. 有IPv4和IPv6。
  5. IP允许添加新的字段到Header。

IPv4 Datagram

Field Size (bit) Comment

Header length 4 指定了 header 的长度(32bit为单位)，通常 options field 是空所以 header 长度一般是 20 bytes，也就是 5 words

Identifier 16 分片和重组
Flags 3 分片和重组
Fragmentation offset 13 分片和重组

Upper-layer-protocol 8 在到达目的主机时使用，指定了应该将数据包交给哪个传输层协议。例如，6 表示给 TCP，17 表示给 UDP。port 是传输层和应用层之间的分用-复用，而这是网络层和传输层的

IPv4地址

netmask：告诉我们哪些IP address是Local address。比如两个IP address A 和 B，if (A & mask == B & mask) then same network。在同一个网络里，就可以直接发送Packet了。比如：

  • 255.255.255.0 means first 24 bits match。
  • 255.128.0.0 means first 9 bits match。
  • 所以越小的netmask代表了一个越大的network。

我们可以用ifconfig查看我们的netmask。

Address Resolution Protocal(ARP)

ARP的用途：Network layer can discover the link address associated with a network address it’s directly connected to.或者说，可以用来解决这样一个问题：我有一个IP packet，接下来我要把他转发到这个IP address，但是他的link address是什么呢？

回顾一下之前的内容，网络是分层的，IP所在的network lawyer下面是link lawyer。他们用的不同的地址。network层基本都是IP协议，都是IP地址。link层，比如说常用的Ethernet，用的是一个48 bit address，也叫Mac地址。

一个基本的ARP过程：

  1. 我Boardcast在整个子网内一个request，谁是IP地址xxx.xxx.xxx.xxx啊？
  2. 拥有对应IP的，单独发送reply（事实上也可以Boardcast，方便刷新局域网内所有的cache）给我，告诉我他的Link address
  3. 我在ARP表内缓存IP address和Link address的对应关系。
