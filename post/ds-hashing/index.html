Hash Table ADT

Hash Table是一组key-value的集合，并且key是唯一的。我们在python中的字典类型就是一个很好的例子。我们通过把Key映射到Table中的一个特定的位置来访问Value，加快查找的速度。
这个映射的方法就是哈希函数hash function。哈希(Hash)算法，更为准确的翻译是散列算法。是一种将字符组成的字符串转换为固定长度（一般是更短长度）的数值或索引值的方法。可以实现常数平均时间执行插入、删除和查找。
一个理想的（但是不可能的）hash function把每个key映射到一个不同的位置。不同的key被映射到同一个位置就是碰撞/冲突(collision)：f(x1)=f(x2) when x1≠x2f(x_1)=f(x_2)\ when\ x_1\neq x_2f(x1​)=f(x2​) when x1​​=x2​ 。解决碰撞问题有不同的策略。
下面我们来看hash function和解决碰撞的策略。

主要实现Delete、Find、Insert三个接口。

Hash Function

哈希函数的要求：

  1. 容易计算
  2. 减少碰撞
  3. 让结果分布更加均匀，最好是均匀分布。

余数法

如果x是一个整数：f(x)=x % TableSizef(x) = x\ \%\ TableSizef(x)=x % TableSize。通常TableSize取成素数。
相应的如果x是一个字符串：f(x)=(∑x[i]) % TableSizef(x)=(\sum{x[i]})\ \%\ TableSizef(x)=(∑x[i]) % TableSize 或 f(x)=(∑x[i]∗Ni) % TableSizef(x)=(\sum{x[i]*N^{i}})\ \%\ TableSizef(x)=(∑x[i]∗Ni) % TableSize，这个N的取值应该是x[i]的范围（当然也不必那么严格），同样的如果用2^n可以用位移代替乘法。

//这是一个非常高效的hash算法
string_hash(const char *Key, int TableSize)
{
    unsigned int HashVal=0;
    while (*Key != 0)
    {
        //不考虑溢出，加速！
        HashVal = (HashVal<<5) +*(Key++);
    }
   	return HashVal % TableSize;
}

解决碰撞

分离链接法 separate chaining

可以把发生碰撞的key都保留在同一个链表中。
为了方便起见在原来的表中存储了头节点。这样我们由key f(key)=indexf(key)=indexf(key)=index之后，再在index所存储的头节点所在的链表中插入我们真正要存储的value。
由此我们也可以看到把元素尽可能均匀的散列的重要性。最坏情况全部冲突就变成列表了。。

当然，不一定非要是链表，也可以是二叉树等别的数据结构。我们下面来看这种方法的Hash Table ADT接口实现。

Position Find(ElementType Key,HashTable H)
{
  Position P;
  List L;
  L = H->TheLists[Hash(Key,H->TableSize)];

  P= L->Next;
  while (P!=NULL && P->Element!=Key)
    P = P->Next;
  return P;	
}

void Insert(ElementType Key,HashTable H)
{
  Position Pos,NewCell;
  List L;
  Pos = Find(Key,H);
  if (Pos == NULL)	//没有找到key，要插入
  {
    NewCell = malloc(sizeof(Node));
    L = H->TheLists[Hash(Key,H->TableSize)];
    NewCell->Key = Key;
    NewCell->Next = L->Next;
    L->Next = NewCell;		//插在最前面简单
  }
}

//Delete中我们可以不真正的删除，而是把它标记为删除，称为Lazy Delete
//当然这么实现的话上面的实现也要对应的改变
void Delete(ElementType Key,HashTable H)
{
  Position P = Find(Key,HashTable H);
  P->empty = 1;		
}

开放定址法 open addressing hashing

如果有冲突发生，那么我们就尝试选择另外的单元。具体的说我们依次尝试位置h0(X),h1(X),h2(X),...h_0(X),h_1(X), h_2(X),...h0​(X),h1​(X),h2​(X),...，其中hi(X)=(Hash(X)+F(i)) mod TableSizeh_i(X)=(Hash(X)+F(i))\ mod\ TableSizehi​(X)=(Hash(X)+F(i)) mod TableSize。关键就是这个函数F(i)F(i)F(i)，他是解决冲突的办法。

最简单的，我们可以让F(i)=iF(i)=iF(i)=i，这个就叫线性探测法 linear probing。但是可能会导致元素扎堆，形成一次聚集(primary clustering)。
进一步我们可以让F(i)=i2F(i)=i^2F(i)=i2，这个叫做平方探测法 quadric probing。可以注意到平方函数可能不能遍历整个表，也就是说可能表还没有填满就放不进去元素了。如果表的大小是素数，我们可以证明表至少有一半是空着的时候可以填入新元素。
如果表的大小是4k+34k+34k+3型的素数并且探测方法为±i2\pm i^2±i2的时候可以保证遍历整个表。

双散列 double hashing

F(i)=i∗hash2(X)F(i)=i*hash_2(X)F(i)=i∗hash2​(X)。我们要好好选择表的大小和hash_2散列函数。一个关键的点是hash_2函数不能算出0（不然就死了）。表的大小一定要是素数，不然可能无法遍历整个表。

再散列 Rehashing

如果表被填的比较满了，那我们解决碰撞的算法再好可能要反复寻找insert的地方，这不好。一个解决办法是建立另一个更大的表并使用一个新的相关的hash function，然后把原来表的元素通过新的hash function散列到新的表里。

一般来说新的更大的表会选择原来的两倍大小（但是表的大小最好是素数），判断何时要进行再散列也有不同的策略。
