  3. 内存用于存储程序中的数据，由存储期，作用域，链接表征。注意对象和标识符的区别。
    生命周期（存储期）：所谓存储期是指“对象”在内存中保留了多长时间。这个的最好理解方式是从汇编的角度考虑。自动存储器都是在进入函数的时候，在栈上动态分配的内存，所以生命周期和函数一样。静态存储（static）的变量分配在内存里，所以和程序有相同的生命周期。

    1. 静态存储期(static)：在程序执行时分配内存，在程序运行整个过程都存在。
      具有文件作用域的变量自动具有（也必须是）静态存储期。
      只会初始化一次，未显式初始化会被初始化成0.
    2. 自动存储期(auto)：程序进入变量定义所在块时分配变量的内存，在程序离开块时释放内存，缺省就是auto
    3. 线程储存期：线程存储期用于并发程序设计，程序执行可被分为多个线程。具有线程存储期的对象，从被声明时到线程结束一直存在。以关键字_Thread_local 声明一个对象时，每个线程都获得该变量的私有备份。
    4. 动态分配存储期：调用malloc()（之类的函数）时分配内存，调用free()释放内存。
      作用域和链接：这个的最好理解方式是从编译器的角度考虑。
      作用域：（“标识符”的作用域和链接表明了程序的哪些部分可以使用它。）
    5. 文件作用域：定义在函数之外,声明之后所有的函数都可见(全局变量)
    6. 块作用域：对该块以及包含的嵌套块可见，变长数组稍有不同，它们的存储期从声明处到块的末尾，而不是从块的开始处到块的末尾。
    7. 函数原型作用域（function prototype scope）用于函数原型中的形参名（变量名），如int mighty(int mouse, double large); 函数原型作用域的范围是从形参定义处到原型声明结束。这意味着，编译器在处理函数原型中的形参时只关心它的类型，而形参名（如果有的话）通常无关紧要。
      链接：
      1. 无链接：块作用域自然是无链接的。
      2. 内部链接：static。只在当前的编译单元（.c文件）中可用。
      3. 外部链接：别的文件需要用extern再声明一遍。
        一些变量的修饰符：
        volatile：告诉计算机代理（而不是程序）可以改变该变量的值
        const：只读，不能修改
        restrict:表明指针是访问数据对象唯一且初始的方式
        extern:只是声明别处已经定义的变量
        regester:寄存器（不算内存）变量，加快运行速度，但是不能乱用
        局部参数如果和全局参数同名会暂时掩盖全局参数，

    标准高级 I/O（standard high-level I/O）使用 C 库的标准包和 stdio.h 头文件定义。
    C 程序会自动打开 3 个文件，它们被称为标准输入（standard input）、标准输出（standard output）和标准错误输出（standard error output）。在默认情况下，标准输入是系统的普通输入设备，通常为键盘；标准输出和标准错误输出是系统的普通输出设备，通常为显示屏。

模式字符串 含义
"r" 以读模式打开文件
"w" 以写模式打开文件，把现有文件的长度截为 0，如果文件不存在，则创建一个新文件
"a" 以写模式打开文件，在现有文件末尾添加内容，如果文件不存在，则创建一个新文件
"r+" 以更新模式打开文件（即可以读写文件）
"w+" 以更新模式打开文件（即，读和写），如果文件存在,则将其长度截为 0；如果文件不存在,则创建一个新文件
"a+" 以更新模式打开文件（即，读和写），在现有文件的末尾添加内容，如果文件不存在则创建一个新文件；可以读整个文件，但是只能从末尾添加内容

"rb"、"wb"、"ab"、"ab+"、"a+b"、"wb+"、"w+b" 与上一个模式类似，但是以二进制模式而不是文本模式打开文件

20.序列点
下面就是标准中定义的顺序点：
（1）函数调用时，实参表内全部参数求值结束，函数的第一条指令执行之前（注意参数分隔符“,”不是顺序点）；
（2）&&操作符的左操作数结尾处；
（3）||操作符的左操作数结尾处；
（4）?:操作符的第一个操作数的结尾处；
（5）逗号运算符；
（6）表达式求值的结束点，具体包括下列几类：自动对象的初值计算结束处；表达式语句末尾的分号处； do/while/if/switch/for语句的控制条件的右括号处；for语句控制条件中的两个分号处；return语句返回值计算结束（末尾的分号）处。

struct ListNode *readlist()
{
    struct ListNode *head=NULL,*new;
    int number;
    loop:
        scanf("%d",&number); 			//data读入
        if (number == -1) return head;	//出口
        new=(ListNode *)malloc(sizeof(ListNode));
        new->data=number;
        insert(head,new);//排序在insert做好
    goto loop;
}

倒置：

struct ListNode *reverse( struct ListNode *head )
{
    if (head=NULL) return head;
    struct ListNode *cur=head->next,*tmp,*before;
    head->next=NULL;
    before=head;
    while (cur!=NULL)
    {
        tmp=cur->next;
        cur->next=before;
        before=cur;
        cur=tmp;
    }
    return before;
}

删除一大堆：

struct ListNode *deletem( struct ListNode *L, int m )
{
    struct ListNode *head=L,*tmp,*before;
    while (head!=NULL && head->data==m)
    {
        tmp=head;
        head=head->next;
        free(tmp);
    } 
    for (tmp=head;tmp!=NULL;)
    {
        if (tmp->next && tmp->next->data==m) tmp->next=tmp->next->next;
        else tmp=tmp->next;
    }
    return head;
}
