DS-Disjoint Set

2020-11-12 Views DS633字3 min read繁

参考资料：《数据结构与算法 C语言版》，https://www.cnblogs.com/morui/p/10791044.html（注意在链接中给的根的定义是find(A)==A，而不是下面所说的find(A)=0）

等价关系

等价关系是自反，传递，对称的。等价关系确定等价类。

并查集，也叫做不相交集，Disjoint set。

定义

typedef int DisjSet[NumSets+1];		
typedef int SetType;	//集合的代表
typedef int ElementType;	//元素

void Initilialize(DisjSet S);	
void Union(DisjSet S,SetType Root1, SetType Root2);		//合并Root1和Root2代表的集合
SetType Find(ElementType X,DisjSet S);		//寻找元素X所在的集合的代表

Operations

Union

Union N Union+M Find时间复杂度
Arbitrary Union 最差一条链表，O(N+MN)
Union-by-Size 最深的节点不会超过log N，所以O(N+Mlog N)
Union-by-Height 最深的节点不会超过log N，所以O(N+Mlog N)
  1. 数组实现1(最简单的方法O(1))：
void Union(DisjSet S,SetType Rt1,SetType Rt2)
{
	S[Rt2] = Rt1;
}
  1. 数组实现2(union-by-size)：
    对于这种实现方法，我们要让S[root] = -size，换句话说我们把这个树的大小以绝对值的形式存储在树根。然后，我么总是让较小的树成为较大的树的子树。
void Union(DisjSet S,SetType Rt1,SetType Rt2)
{
//	Rt1 = Find(Rt1);
//	Rt2 = Find(Rt2);	如果我们以root(Find(A))作为参数，则不需要这条语句
	if (S[Rt1]<S[Rt2])	//|S[Rt1]| > |S[Rt2]| 
		{
            S[Rt1] += S[Rt2];   //更新size
            S[Rt2] = Rt1;   //更新root
	else
		{
            S{Rt2} += S[Rt1];
            S[Rt1] = Rt2;
        }
}
  1. 数组实现3(union-by-height)：
    类似的，我们让深度（高度）较小的树成为高度较大的树的子树。
void Union(DisjSet S,SetType Root1,SetType Root2)
{
	if (S[Root2] < S[Root1])
		S[Root1] = Root2;
	else
	{
		if (S[Root1] == S[Root2])
			S[Root1]--;
		S[Root2] = Root1;
	}
}

Find

Find要求返回一个元素所在集合的代表(root)。其实唯一的要求是如果a,b属于同一集合，那么Find(a)==Find(b)。
数组实现(对于上面三个求Union的方法，分别会给Find不同的时间复杂度)：

SetType Find(ElementType X,DisSet)
{
	while (S[X]>0) X=S[X];
	return X;
}

数组实现(路径压缩)：

SetType Find(ElementType X,DisSet S)
{
	  return x<0 ? x : (root[x]=Find(root[x]));
}

root[x]=x

我们来看另一种并查集的表示方式
root[i]==i;表示根节点

Find

路径压缩算法：

    return x == root[x] ? x : (root[x]=Find(root[x]));

路径压缩算法和Union-By-Size和任意的Union是兼容的，压缩之后并不会改变Size。但是对Union-by-height来说就不是这样了。

Union

void Union(x,y)
{
    root[Find(x)] = Find(y);
}
