本地部署静态页面 Gridea+Docker

2021-03-14 Views Others511字3 min read繁

红黑树不会，浪费时间搭建server blog。

基本配置准备工作

  1. 首先，先从库中Pull ubuntu. docker pull ubuntu
  2. 然后，先从Image产生container，把80端口和22端口映射出来（80作为http访问的端口，22作为ssh的端口）。docker run -it -p 8080:80 -p 2222:22 --name ubuntu_web ubuntu
  3. 对ubuntu进行基本配置
    1. 换成国内的源，比如清华tuna。这个问题不大。
      cp /etc/apt/sources.list /etc/apt/sources.list.bak #备份
      vim /etc/apt/sources.list #修改
      apt-get update #更新列表
    2. 安装常用的包
      apt-get install vim git sudo net-tools python3 python3-pip
    3. 修改系统的一些配置
      passwd 修改密码
      useradd 添加用户
    4. 安装ssh并调整配置
      1. apt-get install ssh 安装。
      2. vim /etc/ssh/sshd_config 修改配置。
      3. 把本机上isa.pub的内容复制到~/.ssh/authorized_keys上。
      4. /etc/init.d/ssh start 启动服务。
      5. 然后我们就可以通过ssh来访问我们的docker了。测试一下ssh -p 2222 username@127.0.0.1
  4. 调整Gridea的配置：reticence->username
  5. 然后test测试一下，成功的话就保存，然后同步。

部署 via Python

至此，我们已经把静态文件都保存到了我们的docker中，怎么部署他们呢？
当然可以用nginx，不过还有一个更简单的方法，就是python。
在静态文件目录/home/username/www下，执行python3 -m http.server 80，创建http服务在80端口，然后再主机上就映射到了8080端口。打开浏览器，访问http://127.0.0.1:8080。看到你的页面，就大功告成了。
可以把之前的命令做成脚本，然后实现一键部署。

Gridea使用技巧

Gridea有内置的文章编辑器，但是最好的markdown编辑器肯定是typora啦。
可以把typora的图片保存路径设置成../post-image，这样图片导入甚至比利用Gridea还方便。
大部分typora的特性Gridea都是支持的，偶尔可能会有解析错误。
注意的是公式符号$和代码中间不要有空格，否则Gridea不能正确解析。
