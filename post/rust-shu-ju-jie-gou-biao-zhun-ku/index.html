感觉Option和Result在Rust中是最常见的元素了，可以用他们来解决错误处理等问题。常常和match语句一起使用，match匹配要求枚举是穷尽的。_可以当做default。match是最通用的匹配方法，但是因为比较繁琐，所以Rust提供了很多的语法糖和方法。

match map_b.get(key){
    Some(num) => a = a + f64::from(value * num),
    None => a=a + 0.0
}
if let Some(x)=optVal{
    doSomethingWith(x);
}

容器

Vec

Vec<T>，对应于C++中的Vector。允许我们在一个单独的数据结构中存储多于一个的、相同类型的、在内存中彼此相邻排列的值。换句话说，是一个大小可变的数组。可以实现数组、队列、栈等线性表数据结构。

重载了Index运算符，所以可以直接用[]取下标的形式访问内部成员。还重载了Deref/DerefMut运算符，所以可以自动的被解引用成为数组切片。

// 创建
let v1: Vec<i32> = Vec::<i32>::new();				// 构造一个空的vector
let v2: Vec<String> = Vec::with_capacity(1000);		// 预先分配空间，减小运行时扩容的开销
let v3 = vec![1,2,3];								// 利用宏来创建vector
let mut v: Vec<i32> = vec![0;100];					// 可变

// 插入数据
v.push(4);							// 插在最后面
v.extend_from_slice(&[1,2,3]);		// 批量插在最后面
v.insert(2,100);					// 指定位置的插入，index的范围是[0,len]
v[2] = 100;							// index的范围是[0,len)

// 访问数据
let third:i32 = &v[2];				// 如果下标越界会产生panic
match v.get(2) {
    Some(third) => ...,
    None => ...,
}									// get 返回Option，配合模式匹配
if let Some(i) = v4.get(6){
    ...
}

//遍历，为了修改可变引用指向的值，必须先使用解引用运算符获取i中的值
for i in &mut v{
    *i += 50;
}

查看Push的源代码：pub fn push(&mut self, value: T) {...}，传递了自己的一个可变引用。然后value的类型是T，也就是说所有权也移动进来了。小心了。

HashMap

HashMap<K, V, S>是基于Hash算法的存储一组键值对的容器。K是键，V是值，S是哈希算法的类型，一般采用默认值。K要求实现了Eq和Hash两个Trait。

如果我们采用自己的类型作为K，就要实现者两个trait。Eq就是离散中的等价关系，一般用自动推导#[derive(PartialEq, Eq, Hash)]，Hash一般采用自动推导#[derice(Hash)]。

HashMap<K, V> 类型储存了一个键类型 K 对应一个值类型 V 的映射。
map 可以用于需要任何类型作为键来寻找数据的情况，而不是像 vector 那样通过索引。

use std::collections::HashMap;
// 创建
let mut scores = HashMap::new();

// 插入
scores.insert(String::from("Blue"), 10);
scores.entry(String::from("Yellow")).or_insert(50);	// 在没有的情况下才插入

// 取值
let n :i32 = 1;
println!("{:?}",scores.get(&n));		// get方法返回的是Option，里面的内容是引用，没有所有权。

// 删除
scores.remove(&p);

// 遍历
for (key, value) in &scores {
    println!("{}: {}", key, value);
}

BTreeMap

BTreeMap<K，V>是基于B树数据结构的存储一组键值 key-value-pair的容器。BTreeMap对key的要求是满足Ord约束，即具备“全序”特征。

BTreeMap和HashMap的接口很像。不过正如B Tree的结构决定的，它可以实现范围查找。

for (k, v) in map.range(2..6) {
	println!("{} : {}", k, v);
}

Iterator

迭代器是Rust的一项重要功能。Rust的迭代器是指实现了Iterator
trait的类型。它最主要的一个方法就是next()，返回一个Option<Item>，所以迭代器一定要有一个成员需要存储上次返回的情况。一般情况next()返回Some(Item)；如果迭代完成，就返回None。

Rust标准库有一个命名规范，从容器创造出迭代器一般有三种方法：

  • iter(): 创造一个Item是&T类型的迭代器；
  • iter_mut(): 创造一个Item是&mut T类型的迭代器
  • into_iter(): 创造一个Item是T类型的迭代器。

for item in container {}

Rust的forin{}语法结构就是一个语法糖。这个语法的原理其实就是调用.into_iter()方法来获得迭代器，然后不断循环调用迭代器的next()方法，将返回值解包，赋值给，然后调用语句块。

类似的还有for item in &container {}和for item in &mut container {}

String

Rust 的核心语言中只有一种字符串类型：str，也就是字符串slice，它通常以被借用的形式出现&str。称作 String 的类型是由标准库提供的，而没有写进核心语言部分，它是可增长的、可变的、有所有权的、UTF-8 编码的字符串类型。Rust 标准库中还包含一系列其他字符串类型，比如 OsString、OsStr、CString 和 CStr。
(也就是说在进行不能使用标准库的系统编程时，你的选择就只有str了)

//创建一个字符串，看源代码的话他们的实现是一模一样的。
let mut s = String::from("lo");
let mut s = "lo".to_string();   
//可以通过 push_str 方法来附加字符串 slice，从而使 String 变长。
let mut s = String::from("foo");
s.push_str("bar");
//拼接字符串
let s1 = String::from("Hello, ");
let s2 = String::from("world!");
let s3 = s1 + &s2; // 注意 s1 被移动了，不能继续使用
let s = format!("{}-{}*{}", s1, s2, s3);    //更好的字符串拼接办法
//遍历字符串的办法，字符串是不能直接索引的。
for c in "नमस्ते".chars() {
    println!("{}", c);
}

fmt

std::fmt

format!("Hello");                 // => "Hello"
format!("Hello, {}!", "world");   // => "Hello, world!"
format!("The number is {}", 1);   // => "The number is 1"
format!("{:?}", (3, 4));          // => "(3, 4)"
format!("{value}", value=4);      // 命名参数
format!("{1} {0}", 1, 2);           // 位置参数
format!("{:04}", 42);             // => "0042" with leading zeros
format!("{:.4},3.14159");        // 保留小数
format!("{:.2$},3.14159,2");        // 保留小数，使用第二个参数作为保留标准

对齐和填充[fill]</>/^

println!("Hello {:0<5}!","X");  //左对齐，填0
println!("Hello {:-^5}!","X");  //居中，填-
println!("Hello {:*>2$}!","X",5);  //右对齐，填*。使用第二个参数作为填充标准。

I/O

std::fs和std::io

在C++里面，标准输入输出流cin、cout是全局变量。在Rust中，基
于线程安全的考虑，获取标准输入输出的实例需要调用函数，分别为
std::io::stdin()和std::io::stdout()。

stdin()函数返回的类型是Stdin结构体。这个结构体本身已经实现了Read trait，所以，可以直接在其上调用各种读取方法。但是这样做效率比较低，因为为了线程安全考虑，每次读取的时候，它的内部都需要上锁。提高执行效率的办法是手动调用lock()方法，在这个锁的期间内多次调用读取操作，来避免多次上锁。

env

在Rust中，进程启动参数是调用独立的函数std::env::args()来得到的，或者使用std::env::args_os()来得到，进程返回值也是调用独立函数std::process::exit()来指定的。
