Vim使用

2021-06-21 Views Tools625字3 min read繁

因为Vim的使用技巧实在是太多了。。所以碰到一个问题百度一次记录一次。

基本概念

下次从网上找一份。

基本使用

i 进入编辑模式 esc返回 :命令输入

/向后查找 ?向前查找

:w 保存 :wq 保存并退出 :q! 不保存退出

复制粘贴

y: copy p: paste

字符串的查找替换

基本的模板是这样的 :[range]s/from/to/[flags]

  • range:搜索范围，如果没有指定范围，则作用于但前行。

    1. :1,10s/from/to/ 表示在第1到第10行（包含第1，第10行）之间搜索替换；
    2. :10s/from/to/ 表示只在第10行搜索替换；
    3. :%s/from/to/ 表示在所有行中搜索替换；

  • flags 有如下四个选项：

    1. c confirm，每次替换前询问；
    2. e error， 不显示错误；
    3. g globle，不询问，整行替换。如果不加g选项，则只替换每行的第一个匹配到的字符串；
    4. i ignore，忽略大小写。

  • from是用来匹配的字符串，to是用来替换的字符串

    • from是可以利用正则表达式进行替换的
    • 在from中以\(和\)括起来的正则表达式，在to使用的时候可以用\1、\2等变量来访问\(和\)中的内容。

例子：

  1. 换data10_w,data11_w,data12_w,data_13_w的末尾字母换成d。:%s/1\([0-3]\)_w/1\1_d/g

  2. 把verilog中的输入输出转换成调用约定:%s/\(input\|output\) \(wire\|reg\).* \(\w*\),/.\3(\3),/g，比如：

    input wire clk, // clock
input wire rst, // reset
input wire [31:0] data_cpu_write,   // CPU要往内存中写的数据
input wire [31:0] data_mem_read,    // 从内存中读取的数据
input wire [31:0] addr_cpu, // cpu addr
input wire wr_cpu, // cpu write enable
input wire rd_cpu, // cpu read enable
input wire ready_mem, // memory ready 因为memory不一定能在一个周期内

output reg wr_mem, // memory write enable
output reg rd_mem, // memory read enable
output reg ready_cache, // cache ready
output reg [31:0] data_mem_write,   // 往内存中写的数据
output reg [31:0] data_cpu_read,    // CPU读取到的数据
output reg [31:0] data_mem_addr,     // 往内存中写数据的地址

--> 

.clk(clk) // clock 
.rst(rst) // reset 
.data_cpu_write(data_cpu_write)   // CPU要往内存中写的数据
.data_mem_read(data_mem_read)    // 从内存中读取的数据 
.addr_cpu(addr_cpu) // cpu addr
.wr_cpu(wr_cpu) // cpu write enable
.rd_cpu(rd_cpu) // cpu read enable
.ready_mem(ready_mem) // memory ready 因为memory不一定能在一个周期内

.wr_mem(wr_mem) // memory write enable
.rd_mem(rd_mem) // memory read enable   
.ready_cache(ready_cache) // cache ready
.data_mem_write(data_mem_write)   // 往内存中写的数据
.data_cpu_read(data_cpu_read)    // CPU读取到的数据 
.data_mem_addr(data_mem_addr)     // 往内存中写数据的地址

格式化

gg=G gg回到开头，G到结尾，=格式化
