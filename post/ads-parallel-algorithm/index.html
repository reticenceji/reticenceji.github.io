2021-06-22 7 min read # DS

PRAM

Parallel Random Access Machine
PRAM 它和 RAM 一样，假设在每个单位时间，每一个 processor 可以做一个计算或者访存的操作。一共 P 个核共享一个 shared memory。

这个模型通常来讲有几种模式（Memory的同步模型）：

Exclusive read exclusive write (EREW) — 多个processor不可以同时读或者同时写同一个内存位置

Concurrent read exclusive write (CREW) — 多个processor 不可以同时写，但是可以同时读同一个内存位置

Exclusive read concurrent write (ERCW) — 不能同时读但是竟然能同时写，这个没什么意义，所以没人用它

Concurrent read concurrent write (CRCW) — 两个processor既可以同时读也可以同时写同一位置，花费单位时间。关于这个同时写最后结果是什么，也有许多说法，不过我们的重点不在这个模型上，就不展开了。

然而这个模型不太现实，一个是高度同步的内存不现实，另一个是在PRAM里，一个算法需要几个 processor，这个 P 是固定的。在我们现代的多核电脑里，虽然你知道它有几个 processor，但是实际上，你不知道你能用几个。

Work-depth

它的中心思想是说，对于我们所有算法中的操作，根据其依赖关系可以画成一张 DAG（Directed acyclic graph，有向无环图）。其中每一个节点代表一个操作，每一条有向边 A->B 意味着B这个操作必须等待A操作进行完才可以执行。我们管这张图叫 computation graph 或者computation DAG。

**一个算法的代价用两个量来衡量，一个叫 work，用 W 表示，也就是这个图总的结点数。它告诉你这个算法如果你只有一个 processor，串行地跑，它的时间复杂度是什么。**比如上图，这个数就是 17。第二就是这个图本身的深度，叫 Depth (D)，也叫 Span (S)，也就是最长的并行依赖链。在上图里，这个数是 8。这个 depth 告诉你，如果你有无数个 processor，你这个算法需要多少时间——因为即便你有无数个 processor，这些依赖关系还是要被按顺序一个等一个地执行的。

可以证明，给定一个 DAG，W 的 work , D 的 depth , p 的 process 你总是可以通过一定的调度算法，在 W/p+O(D) 的时间内给它执行完。

Examples

Prefix-sum

input: a1, a2, a3, ..., an

output: a1, a1+a2, a1+a2+a3, ..., a1+a2+...an

首先我们规定一个符号，(0,α)(0,\alpha)(0,α)是(h,i)(h,i)(h,i)最右边的孩子的prefix-sum。我们要求解的目标其实就是C(0,i)C(0,i)C(0,i)。

先用Parallel算法求出B(h,i)B(h,i)B(h,i)。这是一个自底向上的过程，这个过程很简单。

然后再用Parallel算法求出C(h,i)C(h,i)C(h,i)，这是一个自顶向下的过程。Work = O(n) Time = O(log N)

for (h=log N;h>=0;h--){
    for (i==1) pardo
        C[h,i] = B[h,i];
    for (i%2 == 0) pardo
        C[h,i] = C[h+1,i/2];	// parent
    for (i%2 == 1 && i!=1) pardo
        C[h,i] = C[h+1,(i-1)/2] + B[h,i];
}

Merging

merge two non-decreasing arrays A(1), A(2), …, A(n) and B(1), B(2), …, B(m) into another non-decreasing array C(1), C(2), …, C(n+m)

先定义RANK:

RANK(j, A) = i,   if A(i) < B(j) < A(i + 1), for 1 < i < n
RANK(j, A) = 0,  if B(j) < A(1) 
RANK(j, A) = n,  if B(j) > A(n)

然后我们解决Rank(A,B)，就是知道Rank(i,A) for B和Rank(i,B) for A。

在解决了Rank(A,B)之后，就可以用O(1)的时间和O(n+m)的work解决merging问题了，看一个例子：

	A = 1 3 4	B = 2 5 6
Rank	0 1 1		1 3 3	(Rank in B/A)
Where	0 1 2		0 1 2	(Position in A/B)
Result	0 2 3		1 4 5	(Position in C = Rank+Where)

那么我们就把问题转换成了求Rank的问题。

  1. 第一个办法：对A中的每个元素，在B中用Binary Search求Rank。Time = O(log n). Work = O(nlog n) （这里就当m=n了）。

    // 这两个也是可以并行的
for 1<= i <= n pardo
	Rank(i,B) = BinarySearch(A(i),B);	
for 1 <= i <= m pardo
	Rank(i,A) = BinarySearch(B(i),A);

  2. 第二个办法：串行。 Time = O(n+m) Work = O(n+m)

    i = j = 0; 
while ( i <= n || j <= m ) {
    if ( A(i+1) < B(j+1) )
        RANK(++i, B) = j;
    else RANK(++j, A) = i;
}

  3. 第三个办法：并行。比第一个办法好。Time = O(log n), Work = O(n)

    1. 先选择部分的(p = n/log n)个元素按照第一个办法进行ranking。那么Time = O(log n), Work = O(n)。选择的元素如下所示：

      A_Select(i) = A(1+(i-1)log n)   for 1 <= i <= p
B_Select(i) = B(1+(i-1)log n)   for 1 <= i <= p

    2. 把上面的ranking完成之后，剩下最多有2p个子问题（绿色），每一个子问题的size log n级别。我们按照第二个办法进行Rank，Time = O(log n) Work = O(p log n) = O(n)

Maximum Finding

从n个元素中找到最大的元素

  1. 方法一，两两比较，简单粗暴。Time = O(1) Work = O(n^2)

    for Pi , 1 <= i <= n  pardo
    B(i) := 0
for i and j, 1 <= i,j <= n  pardo
    if ( (A(i) < A(j)) || ((A(i) = A(j)) && (i < j)) )
            B(i) = 1
    else B(j) = 1
for Pi , 1 <= i <= n  pardo
    if B(i) == 0
       A(i) is a maximum in A
