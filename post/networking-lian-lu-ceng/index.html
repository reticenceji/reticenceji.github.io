Networking-链路层

2021-03-07 3 min read # Networking

局域网、广域网
广播、点到点

基本概念

结点node：主机、路由器
链路：网络中两个结点之间的物理通道，比如之前提到的双绞线、光纤。
数据链路：网络中两个结点之间的逻辑通道。把实现控制数据传输协议的硬件和软件加到链路上就构成数据链路。
帧frame：链路层的协议数据单元。

链路层功能概述：

  1. 为网络层提供服务。无确认无连接服务，有确认无连接服务，有确认面相连接服务。
  2. 链路管理，连接的建立、维持、释放
  3. 组帧framing
  4. 流量控制。
  5. 差错纠正和检测（链路层的检测主要是为了提高上层的传输效率）

组装成帧


在IP数据报的前后添加一些字段，帧首部和帧尾部。
首部和尾部包含许多的控制信息，还可以做到定界的作用。

透明传输：有一个可能的问题，就是传输的数据中出现了控制字符（如下图所示）。所以我们要防止这种情况的发生。也就是说，无论数据是什么都应该能正常的传输。有下面几种方法：

  1. 字符计数法
  2. 字符填充法。可以封装的时候，在控制字符前面填充转义字符。（就跟字符串里的""一样），在解的时候，再删除控制字符。
  3. 零比特填充法（ 5"1"1"0"法）：封装的时候遇到连续的5个1就填充一个0。（比如开始和结束的标志是01111110），解的时候遇到连续的5个1就删除后面的0。
  4. 违规编码法：比如曼彻斯特编码只有 “高低”和“低高”，所以我们用“高高”和“低低”作为控制字符。（不是很理解，怎么确定周期呢），IEEE802标准采用。
    就是让原有数据序列中（包括你要加上的一位）1的个数为奇数
    其中零比特和填充法和违规编码法常用。

差错纠正

帧错 ，比如说有帧1,2,3。
丢失：1-3
重复：1-2-2-3
失序：1-3-2

检测错误的基本思想，是添加一些冗余数据。

  1. 奇偶校验码（检错编码）
  2. 循环冗余码CRC（检错编码）
  3. 海明码（纠错编码）

奇偶校验，n-1位信息元+1位校验元。让校验元=原有数据序列中（包括你要加上的一位）1的个数为奇数个。只能检测出奇数比特错误。
