Rust程序的调试

2021-08-07 Views1799字11 min read繁

环境

Rust+VSCode。VSCode安装插件CodeLLDB。
然后点击调试的时候，会自动生成调试配置文件。一般来说也够用了。

{
    // Use IntelliSense to learn about possible attributes.
    // Hover to view descriptions of existing attributes.
    // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [      
        {
            "type": "lldb",
            "request": "launch",
            "name": "Debug executable 'lc3asm'",    // 可执行文件的调试
            "cargo": {
                "args": [
                    "build",
                    "--bin=lc3asm",
                    "--package=lc3asm"
                ],
                "filter": {
                    "name": "lc3asm",
                    "kind": "bin"
                }
            },
            "args": ["test.asm"],       // 程序的参数如果需要的话在这里写
            "cwd": "${workspaceFolder}"     // 路径环境变量
        },
        {
            "type": "lldb",
            "request": "launch",
            "name": "Debug unit tests in executable 'lc3asm'",      //单元测试的调试
            "cargo": {
                "args": [
                    "test",
                    "--no-run",
                    "--bin=lc3asm",
                    "--package=lc3asm"
                ],
                "filter": {
                    "name": "lc3asm",
                    "kind": "bin"
                }
            },
            "args": [],
            "cwd": "${workspaceFolder}"
        }
    ]
}

C/C++

顺便再提一下他们怎么调试。因为没有cargo提供的构建信息，所以可以自己用Makefile/CMake构建出带-g调试选项的可执行文件，然后launch.json选择最朴素的LLDB配置就可以了。

{
    // Use IntelliSense to learn about possible attributes.
    // Hover to view descriptions of existing attributes.
    // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "type": "lldb",
            "request": "launch",
            "name": "Debug",
            "program": "${workspaceFolder}/student_2.out",
            "args": [],
            "cwd": "${workspaceFolder}"
        }
    ]
}
