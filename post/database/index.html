导论 Introduction

什么是数据库DB？

  • a large collection of integrated and persistent data.
  • 长期存储在计算机内部，有组织的，可共享的数据集合。

什么是数据库管理系统DBMS？
DBMS=Database + a set of programs used to access, update and manage database.

数据库管理系统的特性：

  • 数据访问的高效Efficiency和可扩展性scalability
  • 缩短应用开发时间
  • 数据独立性independence（物理数据独立性 / 逻辑数据独立性）
  • 数据完整性integrity和安全性
  • 并发Concurrent访问和鲁棒性（恢复）

View of data

我们用下面的抽象层次结构来管理数据：

  • Physical level：最底层，关注数据是怎么样真实的存储的。
  • Logical level：关注数据是怎么在数据库中存储的，DBA管理的。
  • View level：是logical level存储的数据的一部分，提取出来方便使用。

Schemas and instances，就像类和实例一样，前者是后者的模板。

Logical schemas是最重要的。在关系型数据库中他就是一张表。
Physical schemas是描述底层设计的。

数据模型帮助我们设计数据库：

  • Entitu-Relationship Model ⭐️ ：
  • Relational Model
  • Object-Based Data model

数据库设计

  1. 需求分析
    – 需要什么样的数据、应用程序和业务
  2. 概念数据库设计
    – 使用 E-R 模型或类似的高层次数据模型，描述数据
  3. 逻辑数据库设计
    – 将概念设计转换为某个DBMS所支持的数据模型
  4. 结构优化
    – 关系标准化，检查冗余和相关的异常关系结构
  5. 物理数据库设计
    – 索引，集群和数据库调优
  6. 创建并初始化数据库&安全设计
    – 加载初始数据，测试
    – 识别不同的用户及他们的角色

Database Language

  • DDL=Data Definition Language
  • DML=Data Manipulation Language
  • DCL=Data Control Language

我们学习的重点是SQL。

Database用户

DBA：Database Administrator。拥有最高权限，有下面5个功能：

  • Schema definition. The DBA creates the original database schema by executing a set of data definition statements in the DDL.
  • Storage structure and access-method definition.
  • Schema and physical-organization modification：The DBA carries out changes to the schema and physical organization to reflect the changing needs of the organization, or to alter the physical organization to improve performance
  • Granting of authorization for data access. By granting different types of authorization, the database administrator can regulate which parts of the database various users can access. The authorization information is kept in a special system structure that the database system consults whenever someone attempts to access the data in the system
  • **Routine maintenance.**日常维护

Transaction Management

我们需要让事务具有如下特性：

  • atomicity 原子性：一个事务要么发生，要么不发生
  • consistence 一致性
  • isolation 隔离性
  • durability 持久性：一旦执行成功就会永久保存

二、关系模型 Relational Model

Relation：

  • A relational database is a collection of one or more relations, which are based on relational model. 关系型数据库是使用最广泛的数据库。
  • A relation is a table with rows and columns. 简单的理解Relation就是一张表。
  • The set of allowed values for each attribute is called the domain (域) of the attribute. 简单的理解Attribute就是列，对应的数据取值范围就是Domain。
  • tuple代表的就是数据库中的一行。

Schema & instance：

  • The relation schema describe the structure of the relation. 比如Instructor-schema = (ID: string, name: string, dept_name: string, salary: int)，介绍了表头以及域（类型）。规定了表的结构。
    • 我们用R表示schema. R={ID, name, dept_name, salary}
  • The relation instance corresponds to the snapshot of the data in the relation at a given instant in time.
    • 我们用r(R)表示某个instance。是一张具体的表。他和schema的关系，就跟class和instance一样。

Key：

  • K is a superkey (超码) of R if values for K are sufficient to identify a unique tuple of each possible relation r(R) 。或者说K是可以唯一确定某条记录的，一个或多个属性的集合，
    • K is a candidate key (候选码) if K is minimal superkey. 如果K的任意一个真子集都不是superkey,那么K就是candidate key。这也是我们通常关心的。
    • K is a primary key (主码), if k is a candidate key and been defined by user explicitly. Primary key is usually marked by underline. Primary key是被数据库设计者专门选出的candidate key。我们通常用下划线表示它。
  • 如果一个relation r1中包含另一个relation r2的primary keyKK。那么在r1中，我们把KK称为foreign key。把r1称为refering relation(参照关系)，把r2称为referenced relation (被参照关系).
    • 参照关系中外键的值必须在被参照关系中实际存在或为null。定义外键的好处是防止r(A,B,C)中的B出现无效数据，这种约束称为referential integrity constraint。
