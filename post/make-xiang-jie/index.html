我不想详细的去讲解Makefile的规则。只想记录一下过程中遇到的问题和对应的解决办法。

头文件的位置

gcc -I可以指定头文件的寻找位置。所以可以通过修改CFLAG来修改寻找头文件的目录。

变量的传递

这是最上层的Makefile定义的一些变量，以及编译子文件夹的命令（节选）：

export
INCLUDE = -I ../include
CF = -O0 -march=$(ISA) -mabi=$(ABI) -mcmodel=medany -ffunction-sections -fdata-sections -nostartfiles -nostdlib -nostdinc -static -lgcc -Wl,--nmagic -Wl,--gc-sections -g3
CFLAG = ${CF} ${INCLUDE} -DPRIORITY

（第一个编译目标）
subsystem:
	$(MAKE) -C lib
	$(MAKE) -C init
	$(MAKE) -C arch/riscv

Variable values of the top-level make can be passed to the sub-make through the environment by explicit request. These variables are defined in the sub-make as defaults, but they do not override variables defined in the makefile used by the sub-make unless you use the ‘-e’ switch (see Summary of Options).

If you want to export specific variables to a sub-make, use the export directive, like this:

export variable …

If you want to prevent a variable from being exported, use the unexport directive, like this:

unexport variable …

If you want all variables to be exported by default, you can use export by itself:

export

除了MAKEFLAGS以外的变量默认是不向子目录传递的，除非用了export指令。虽然没有在文档中找到，但是传递到子目录之后的变量就是一整个了，比如我在子目录Makefile中修改了INCLUDE不会对CFLAG造成影响。

$(MAKE) -C subdir 等价于 cd subdir && $(MAKE)

赋值

Makefile中一共有4种赋值符号。

  • =：make会将整个makefile展开后，再决定变量的值。也就是说，变量的值将会是整个makefile中最后被指定的值。看例子，在下例中，y的值将会是 xyz bar ，而不是 foo bar 。：
x = foo
y = $(x) bar
x = xyz
  • :=：表示变量的值决定于它在makefile中的位置，而不是整个makefile展开后的最终值。如果吧上面的=换成:=，那么y的值将会是foo bar。
  • ?=：是如果没有被赋值过就赋予等号后面的值
  • +=：是添加等号后面的值
