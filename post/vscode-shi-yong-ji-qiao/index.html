Debug（还是GDB好用）

Debug的配置由.vscode/launch.json(准确的说，这个JSON是被VSCode扩展的JSON)文件管理。当你打开调试面板的时候，可以选择launch.json里面不同的config进行调试。下面是我的两个，分别是gdb和C/C++插件提供的。如果你的工作目录下没有这个文件，点击调试之后会默认生成一个。以C++为例：

{
    // Use IntelliSense to learn about possible attributes.
    // Hover to view descriptions of existing attributes.
    // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "type": "by-gdb",
            "request": "attach",
            "name": "Attach(gdb)",
            "program": "a.out",
            "cwd": "${workspaceRoot}",
            "processId": 1234
        },
        {
            "name": "gcc - 生成和调试活动文件",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "setupCommands": [
                {
                    "description": "为 gdb 启用整齐打印",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ],
            "preLaunchTask": "C/C++: gcc 生成活动文件",
            "miDebuggerPath": "/usr/bin/gdb"
        }
    ]
}

不过我感觉vscode的调试还是不如直接使用gdb来的方便。另外，debug不方便和gef插件共用。Debug Terminal不能显示非ASCII字符。不知道怎么解决，还是用GDB吧。

VSCode 命令

Ctrl+P可以调出命令窗口。然后命令的不同前缀有不同的含义。前面两个比较重要。

前缀 含义
? 显示可执行的命令
> 输入命令
@ 查找当前工程下的符号
# 查找当前文件下的符号
: 跳转到输入的行号
无 跳转到输入的文件名

Tasks

这个感觉有一点像脚本，不过是用JSON来写的简单脚本。

  • type： 脚本类型
  • label：标签，task的名字
  • command：要执行的命令
  • args：命令的参数。内置的变量可以用ctrl+space来查看提示。
  • options：
  • problemMatcher：
  • groups：任务分组，在build组的任务们，可以通过在命令行输入>run build task来运行，如果任务分组是test，你就可以用>run test task来运行
{
    "tasks": [
        {
            "type": "cppbuild",
            "label": "C/C++: g++ build active file",
            "command": "/usr/bin/g++",
            "args": [
                "-g",
                "*.cpp",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "options": {
                "cwd": "${workspaceFolder}"
            },
            "problemMatcher": [
                "$gcc"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "detail": "Task generated by Debugger."
        }
    ],
    "version": "2.0.0"
}
