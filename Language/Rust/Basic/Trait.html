Reticence's Notes

Trait

What is Trait

Trait中可以定义方法（还有类型），我们可以为类型impl某个Trait，也就是实现Trait中定义的东西。

所有的Trait中都有一个隐藏的类型Self，代表当前实现了这个Trait的具体类型。

trait本身既不是具体类型，也不是指针类型。他只是定义了针对类型的、抽象的约束。不同的类型可以实现同一个trait，满足同一个trait的类型可以具有不同的大小。所以trait在编译阶段没有固定大小，被称为动态大小类型Dynamic Size Type。

Trait也可以有集成关系。比如标准库里的Ord，就继承自Eq和PartialOrd。也就是说，为实现Ord的同时也必须实现Eq和PartialOrd。

#![allow(unused)]
fn main() {
pub trait Ord: Eq + PartialOrd<Self> {
    fn cmp(&self, other: &Self) -> Ordering;

    fn max(self, other: Self) -> Self { ... }
    fn min(self, other: Self) -> Self { ... }
    fn clamp(self, min: Self, max: Self) -> Self { ... }
}
}

Orphan rule

在声明和实现trait的时候，Rust规定了一个一致性规则，也被称为孤儿规则：impl块要么和trait的声明在同一个crate中，要么和类型的声明在同一个crate中。

如果不这样的话，确实让人有点迷惑。

Derive

编译器可以帮我们自动实现某些Trait。

Trait Object

我们有泛型可以帮我们实现静态的分派，有Trait帮我们实现动态的分派。

如果我们希望函数的参数可以是实现了某个Trait的所有类型，我们有两种写法

#![allow(unused)]
fn main() {
fn test<T: XXX>(arg: T) {
    todo!()
}
fn test(arg:dyn XXX ) {
    这样是不对的写法
}
fn test(arg: Box<dyn XXX>) {
    todo!()
}
}

第一种类似于模板，会为每个具体的T类型生成一份代码。效率高，但是二进制会膨胀。第二种是不正确的写法，因为dyn XXX 的大小不定，我们需要用Box把他包起来，这就是Trait Object。

实际上，指向Trait的指针都是trait object。dyn Trait是一个动态大小的类型DST。&dyn Trait, &mut dyn Trait, Box<dyn Trait>, * const dyn Trait, Rc<dyn Trait>等都是Trait object，因为dyn Trait是DST，所以这些指针也应该是fat pointer(携带了长度信息)。就现在来说，对DST的使用基本都是通过其指针完成的，比如函数参数就不能使DST。

另外，这个Trait中的方法第一个参数应该是self，如果不是的话，

Read/Write

#![allow(unused)]
fn main() {
pub trait Read {
    fn read(&mut self, buf: &mut [u8]) -> Result<usize>;

    fn read_vectored(&mut self, bufs: &mut [IoSliceMut<'_>]) -> Result<usize> { ... }
    fn is_read_vectored(&self) -> bool { ... }
    unsafe fn initializer(&self) -> Initializer { ... }
    fn read_to_end(&mut self, buf: &mut Vec<u8>) -> Result<usize> { ... }
    fn read_to_string(&mut self, buf: &mut String) -> Result<usize> { ... }
    fn read_exact(&mut self, buf: &mut [u8]) -> Result<()> { ... }
    fn by_ref(&mut self) -> &mut Self
    where
        Self: Sized,
    { ... }
    fn bytes(self) -> Bytes<Self>
    where
        Self: Sized,
    { ... }
    fn chain<R: Read>(self, next: R) -> Chain<Self, R>
    where
        Self: Sized,
    { ... }
    fn take(self, limit: u64) -> Take<Self>
    where
        Self: Sized,
    { ... }
}
pub trait Write {
    fn write(&mut self, buf: &[u8]) -> Result<usize>;
    fn flush(&mut self) -> Result<()>;

    fn write_vectored(&mut self, bufs: &[IoSlice<'_>]) -> Result<usize> { ... }
    fn is_write_vectored(&self) -> bool { ... }
    fn write_all(&mut self, buf: &[u8]) -> Result<()> { ... }
    fn write_all_vectored(&mut self, bufs: &mut [IoSlice<'_>]) -> Result<()> { ... }
    fn write_fmt(&mut self, fmt: Arguments<'_>) -> Result<()> { ... }
    fn by_ref(&mut self) -> &mut Self
    where
        Self: Sized,
    { ... }
}
}

虽然上面列出了一大堆方法，但是除了read/write/flush以外都是有默认实现的。Read为读取字节流提供了一个抽象，Write为可写入字节流的东西提供了一个抽象。（Unix设计哲学 Everything is file, file is a byte of stream）

  • read: Pull some bytes from this source into the specified buffer, returning how many bytes were read.
  • write: Write a buffer into this writer, returning how many bytes were written.
  • flush: Flush this output stream, ensuring that all intermediately buffered contents reach their destination.

std::error::Error

#![allow(unused)]
fn main() {
pub trait Error: Debug + Display {
    fn source(&self) -> Option<&(dyn Error + 'static)> { ... }
    fn backtrace(&self) -> Option<&Backtrace> { ... }
    fn description(&self) -> &str { ... }
    fn cause(&self) -> Option<&dyn Error> { ... }
}
}

如果你不想处理错误，想用?表达式直接一把梭，那就把返回值写成Result<(), Box<dyn Error>>，或者利用Anyhow这个第三方库。因为标准库所有的Error都是满足这个Trait的。当然，大工程不提倡这种鸵鸟做法，但小工程是写起来是真的爽。

Default

为某个结构提供默认值。

#![allow(unused)]
fn main() {
pub trait Default {
    fn default() -> Self;
}
}

有什么用呢？在填充某些字段的时候可以用default()，很省事。不过这个特性还是nightly的。你需要打开开关并导入default。

#![feature(default_free_fn)]
use std::default::default;

struct Fast {
    v: u8,
    max: u8,
}
impl Default for Fast {
    fn default() -> Self {
        Fast {v: 1, max: 10 }
    }
}
fn main() {
    let f = Fast{v:2, ..default()};
    let mut f2 = Fast{..default()};
}

std::convert::From/std::convert::Into

#![allow(unused)]
fn main() {
pub trait From<T> {
    fn from(T) -> Self;
}
pub trait Into<T> {
    fn into(self) -> T;
}
}

做类型转换。value to value，AsRef是reference to reference。是会消耗所有权的。实现了From，会自动实现Into，From<T> for U implies Into<U> for T，也就是说如果我们不应该去实现Into而是应该去实现From。

另外需要注意的是，这两个方法是不应该失败的。如果可能失败，有TryFrom/TryInto。

这个Trait非常适合用来写泛型代码，为一系列类型做统一抽象。

fn is_hello<T: Into<Vec<u8>>>(s: T) {
   let bytes = b"hello".to_vec();
   assert_eq!(bytes, s.into());
}

fn main() {
    let s = "hello".to_string();
    is_hello(s);
}

std::convert::AsRef/std::convert::AsMut

#![allow(unused)]
fn main() {
pub trait AsRef<T> 
where
    T: ?Sized, 
{
    fn as_ref(&self) -> &T;
}
pub trait AsMut<T> 
where
    T: ?Sized, 
{
    fn as_mut(&mut self) -> &mut T;
}
}

这两个方法差不多。一个是把reference to reference, 一个是把mutable ref to mutable reference，做类型转换。需要注意的是，这种转换的代价必须是非常低的（人家以as打头，命名规范要求零开销）。如果说转换不是零开销的，那么我们可以考虑实现impl From<&T> for &U来实现引用类型的相互转换。

比如，我们看Vec的AsRef<T>实现，这是把&Vec<T>转换成&[T]。是零开销的。他们在内存里实际上是一个东西，都是一个长度+一个指针。

#![allow(unused)]
fn main() {
impl<T, A: Allocator> AsRef<[T]> for Vec<T, A> {
    fn as_ref(&self) -> &[T] {
        self
    }
}
}

这个Trait也非常适合用来写泛型代码，为一系列类型做统一抽象。比如下面这个例子，他可以接受任何类型，只要能被转换成&[u8]即可。

#![allow(unused)]
fn main() {
fn iter_bytes<T: AsRef<[u8]>>(arg: T) {
    for i in arg.as_ref() {
        println!("{}",i);
    }
}
}

std::borrow::Borrow/std::borrow::BorrowMut

#![allow(unused)]
fn main() {
pub trait Borrow<Borrowed: ?Sized> {
    fn borrow(&self) -> &Borrowed;
}
}

这是一个泛型Trait。他做的事情很简单：实现这个trait的类，可以用过borrow方法将它的引用类型转换成其他的引用类型。举个例子，&String类型就可以通过borrow方法变成&str类型。看到这里，回顾一下AsRef，会发现他们不是长得一模一样吗？不过是泛型参数换了一个名字而已。不过他们并不是同一个意思，参考AsRef、Deref和Borrow有哪些区别？ - Ad Infinitum的回答 - 知乎 ，当T实现Borrow<U>时，为T实现Eq, Ord and Hash的时候应与U的实现相同。Borrow常是用于一个类型的不同表示类型(如Box<T>, Rc<T>，Cow<T>)的引用到原类型(T)引用的转换。

#![allow(unused)]
fn main() {
impl Borrow<str> for String {
    #[inline]
    fn borrow(&self) -> &str {
        &self[..]
    }
}
}

这个Trait的用武之地，在于去写一个泛型函数，看文档中提供的例子。这里的check函数，接受的参数限制为 可以通过borrow方法转换成&str 的任何类型。

use std::borrow::Borrow;

fn check<T: Borrow<str>>(s: T) {
    assert_eq!("Hello", s.borrow());
}
fn main() {
    let s = "Hello".to_string();
    check(s);

    let s = "Hello";
    check(s);    
}

另外，Rust为所有的&T和T类型都实现了Borrow。完整的参考文档。

  • T的borrow参数类型就是&T，返回自己即可，这是显而易见的。
  • &T的borrow参数类型是&&T，用两个*获得T，再返回&T。
#![allow(unused)]
fn main() {
impl<T: ?Sized> const Borrow<T> for T {
    fn borrow(&self) -> &T {
        self
    }
}
impl<T: ?Sized> const Borrow<T> for &T {
    fn borrow(&self) -> &T {
        &**self
    }
}
}

BorrowMut和Borrow是类似的，不过是从&T变成&mut T。

Deref/DerefMut

解引用操作可以被自定义。

Rust提供自动解引用机制，所以我们操作智能指针非常的舒服，就像操作普通的指针一样。

std::ops::Drop

#![allow(unused)]
fn main() {
pub trait Drop {
    #[stable(feature = "rust1", since = "1.0.0")]
    fn drop(&mut self);
}
}

类似于C++的析构函数。当拥有所有权的变量结束了生命周期，就会调用。通过这个Trait的实现，让我们可以利用Rust的所有权机制更好的实现资源的管理。在大多数情况下我们不需要自己去实现Drop，因为如果我们自定义了一个struct，他会在释放的时候自动去调用每一个字段的drop函数。

可以用forget主动跳过析构函数。

Clone/Copy

Copy和Clone好像是一个意思？其实Copy只是一个标记，如果一个类实现了Copy，那么编译器认为可以通过简单的内存复制（C语言的memcpy）实现该类型的复制，不会产生任何的内存安全问题。一旦一个类型实现了Copy，那么在变量绑定，函数参数传递，函数返回值传递等场景下，都是copy语义，不再是默认的move语义。只有我们类中每一个字段都实现了Copy，才可以实现Copy，#[derive(Copy)]即可，他只是一个标记。

顾名思义Clone就是克隆一份一样的。如果我们的类中每个字段都实现了Clone，就可以让编译器#[derive(Clone)]。我们也可以自己实现Clone。但是实现的时候要注意：

  1. 对于实现了Copy的类型，那么他的Clone方法应该和copy语义相容，等于按字节复制。遵守这个规则的最好办法就是#[derive(Copy, Clone)]。
  2. clone方法用于按语义的复制。比如Rc类型的clone就是将引用计数+1。

std::marker::Sized

实现是空，是一个标记。实现了这个Trait类型，表示编译期有固定的大小。一般来说，这都是编译器自动隐式实现的，我们不用自己去为我们的类型实现Sized。

需要注意的是，当我们写一个泛型参数的时候，比如struct Foo<T>，实际上默认是struct Foo<T: Sized>。编译器会默认泛型参数T满足Sized。如果我们不希望这样，需要特别写出struct Foo<T: ?Sized>，表示T是否实现Sized都无所谓。

不过在trait中的Self类型（也就是实现这个trait的类型），是默认Self: ?Sized的。并且如果我们用Sized去约束了Self，那么就无法实例化trait object。这是显然的，trait object在运行时刻才知道他具体是哪个类型，才知道他的大小。

#![allow(unused)]
fn main() {
trait Foo where Self: Sized {
    fn foo(&self)
}
}
