Rust-精髓

所有权

Each value in Rust has a variable that's called its owner. There can only be one owner at one time. When the owner goes out of scope, the value will be dropped.

所有权解决了什么问题

通过保证同时只有一个owner，就不会出现反复释放同一块内存（Drop）的情况。防止内存泄露。并且Owner在离开作用域的时候被自动Drop，无需手动。在Rust中没有手动释放内存的操作（有的，不过那个实现很有意思。也不能算是手动释放内存，应该说是提前结束变量的生命周期。

生命周期

https://learning-rust.github.io/docs/c3.lifetimes.html

生命周期解决了什么问题

当我们使用Reference的时候，我们要保证referencing data的生存时间比我们的reference长，准确的说是我们不再使用reference的时候referencing data依然存在。否则就是悬垂引用（在C/C++语言中，就是一个指向无效地址的指针）。生命周期就是为了解决这个问题。

  • A resource can only have one owner at a time. When it goes out of the scope, Rust removes it from the Memory.
  • When we want to reuse the same resource, we are referencing it/ borrowing its content.
  • When dealing with references, we have to specify lifetime annotations to provide instructions for the compiler to set how long those referenced resources should be alive.
  • But because of lifetime annotations make the code more verbose, in order to make common patterns more ergonomic, Rust allows lifetimes to be elided/omitted in fn definitions. In this case, the compiler assigns lifetime annotations implicitly.
#![allow(unused)]
fn main() {
//error:`y` does not live long enough
{
    let x: &Vec<i32>;
    {
        let y = Vec::new();//-|--+
//                               |              |
    }// <+              | x's lifetime
    println!("x's length is {}", x.len());//    |
}// <-+
}

生命周期标记

一篇写的不错的blog。对于普通的变量生命周期很好理解，用C语言的知识就足够了，二者还是类似的。不过考虑到跨函数引用的生命周期，问题就不那么简单了。

这个是一个比较难以理解的地方。首先我们应该知道reference的本质是单纯的指针，无论是lifetime还是mut都是编译器做静态分析时候用的。

每一个reference都应该有一个生命周期标记。一种特殊的参数是'static表示静态变量有和程序一样长的生命周期；一般的参数'a 'b等来表示。'a:'b表示a的生命周期包含b。

对于下面两个'a的含义并不是s2和返回值的生命周期相同，而是使用'a标注的返回的reference借用自具有相同标注的参数。对于下面这个函数，为什么编译器不能帮我们自动推断生命周期呢？可以，但是不好。因为否则的话我们必须阅读源代码才能知道返回值是和s1还是和s2的传入参数借用自具有相同标注的参数。

fn print_ret<'a>(s1: &str, s2: &'a str) -> &'a str {
    println!("s1 is {}", s1);
    s2
}
fn main() {
    let some_str: String = "Some string".to_string();
    let other_str: String = "Other string".to_string();//-+
    let ret = print_ret(&some_str, &other_str);//--+-----+

        x
    } else {

引用 reference

引用的本质是指针，不会获得引用变量的ownership。比如在函数传递参数、for循环枚举的时候我们可能需要这个特性。如下例：

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);

    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

reference:& mutable reference:&mut , dereference:*。 变量 s 有效的作用域与函数参数的作用域一样，不过当引用离开作用域后并不丢弃它指向的数据（不Drop），因为我们没有所有权。当函数使用引用而不是实际值作为参数，无需通过返回值来交还所有权，因为就不曾拥有所有权。 注意一个引用的作用域从声明的地方开始一直持续到最后一次使用为止。 Reference:

At any given time, you can have either one mutable reference or any number of immutable references. （共享不可变 ，可变不共享） References must always be valid. （在Rust中编译器确保引用永远也不会变成悬垂状态：当你拥有一些数据的引用，编译器确保数据不会在其引用之前离开作用域。variable live longer than its reference）

Rust强大的地方在于，他甚至能在编译的时候就保证这些特性。

另一个Slice概念，也可以不是Owner，引用变量的内容，不过是引用集合中一段连续的元素序列。Slice也是不可变的。Slice:s[1..4]。 类似的，Rust在编译的时候，就保证Slice是始终有效的。

我们可以自定义Deref以及DerefMut方法。不过需要注意的是他们的返回类型是reference。也就是说，比如我们对类型String定义了Deref方法返回&str，s的类型是String，那么*s其实是执行了* s.deref()。

所以如果我们对一个类型实现了Deref方法，他就能表现的像一个指针一样或者我们称他为智能指针，比如String我们就可以把它当成一个指针指向一个堆上分配的unicode字符数组，并携带有缓存空间总大小和字符串长度的metadata。

而编译器的自动解引用机制能让我们的代码写的更舒服。什么是自动解引用，就是比如s.len()找不到方法的时候，编译器会自动帮我们看看s.deref().len有没有，s.deref().deref().len有没有......。直到找到为止。另外，.运算符也是可以帮我们自动取引用的。总之，他会帮我们找到正确的方法。

pub trait Deref {
    type Target: ?Sized;
    #[must_use]
    fn deref(&self) -> &Self::Target;
}

pub trait DerefMut: Deref {
    fn deref_mut(&mut self) -> &mut Self::Target;
}

智能指针

Rust 中最常见的指针是第四章介绍的 引用（reference）。引用以 & 符号为标志并借用了他们所指向的值。除了引用数据没有任何其他特殊功能。它们也没有任何额外开销，所以应用得最多。 智能指针（smart pointers）是一类数据结构，他们的表现类似指针，但是也拥有额外的元数据和功能。 我们看的到的String和Vec都是智能指针。通常用结构体来实现，但是智能指针区别于常规的结构体的显著特性在于实现了Deref和Drop两个trait。 Deref允许智能指针结构体实例表现的像引用一样。Drop允许我们自定义当智能指针离开作用域时运行的代码（类似析构函数的感觉）。

Rust中提供了std::rc::Rc<T>类型和std::sync::Arc<T>类型，后者的引用计数是原子操作可以用在多线程中。Rc允许我们有多个所有者指向同一块内存，通过引用计数来管理。（在Rust中这一般是不被允许的）然后当最后一个所有者死亡之后，这一块内存被释放。他和一般的reference相比，最重要的区别是当引用计数减小到0的时候会释放内存，而一般的reference不会负责内存的释放。这个特性就是DropTrait中定义的。然后通过DerefTrait和自动解引用的特性，让我们用起来就和普通的指针一样。

泛型

在Rust中使用泛型，可以让代码适应更多的情况。但是需要注意的是，其实泛型在编译的时候会变成一个确定的类型。 当使用泛型定义函数时，本来在函数签名中指定参数和返回值的类型的地方，会改用泛型来表示。 一个例子：fn largest<T>(list: &[T]) -> T {...} 在结构体重定义泛型。注意下面的integer和float是两个不同的类型。虽然他们都是Point，但是因为Rust 通过在编译时进行泛型代码的 **单态化（monomorphization）**来保证效率，他们实际上被编译成了两种不同的类型。

struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    let integer = Point { x: 5, y: 10 };
    let float = Point { x: 1.0, y: 4.0 };
}

Trait

Trait和别的语言里的implement很像，但是不太一样。 定义一个trait叫做Summary，要实现这个trait，你需要实现一个summarize方法。必须以self（或者说不需要参数，但是要写出来）作为参数并返回一个String。一个trait也可以有多个方法。如下所示：

#![allow(unused)]
fn main() {
pub trait Summary {
    fn summarize(&self) -> String;
}
}

然后你就可以在你自己的类上实现这个trait。一旦实现了 trait，我们就可以用与 NewsArticle 和 Tweet 实例的非 trait 方法一样的方式调用 trait 方法了。如下所示：

#![allow(unused)]
fn main() {
impl Summary for MyClass {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}
}

利用trait，我们可以接受各种各样的参数。比如下面这个例子。本来我们在item: 后面需要书写我们传递参数的具体类型，但是我们现在用impl Summary，那么我们传入的参数可以是任何实现了trait Summary的任何类型。

pub fn notify(item: impl Summary) {
    println!("Breaking news! {}", item.summarize());
}
//需要同时实现两种Trait
pub fn notify(item: impl Summary + Display) {}
//语法糖，指定泛型T和U为满足where从句限定trait的类型。
fn some_function<T, U>(t: T, u: U) -> i32
    where  T: Display + Clone, U: Clone + Debug
{}

少见一点的，你可以把返回值也用trait限定。fn returns_summarizable() -> impl Summary {}，这样的话你需要返回一个实现了trait的类。

一个很典型的例子就是格式化输出。在 println("{}\n",myinstance); 的时候，其实{}就被替换成了执行trait之后的内容。准确来说，是这样子： 标准库中有一个trait叫做std::fmt::Display，里面定义了一个方法签名fn fmt(&self, f: &mut Formatter<'_>) -> Result;。如果你为你自己的类实现了这个方法，那么你就可以用格式化字符串了，或者简单的说就可以用println("{}")了。
