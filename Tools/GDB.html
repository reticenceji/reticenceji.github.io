GDB

Doc,Gef-Doc

在gdb中查看命令帮助的方法是help command

插件安装

修改~/.gdbinit文件：

# 注释掉你不想用的插件
source /data/home/reticence/AAA/gdb/gdbinit-gef.py
#source /data/home/reticence/AAA/gdb/peda/peda.py
  • gef是一个优秀的GDB插件，帮助我们使用GDB进行逆向和漏洞挖掘。他支持多种架构。 安装非常简单，一般的配置都是Linux默认带有的。你可以按照官网的安装脚本，也可以复制gef.py的源代码到你本地的一个文件，然后在.gdbinit里加上就好了。
  • peda，也是一个强力的帮助逆向的GDB插件，不过支持的架构少。安装的话直接git clone https://github.com/longld/peda.git，然后在.gdbinit里加上peda.py就好了。

裸机程序调试指南

首先我们要搞清楚，调试信息和符号表。我们在用gcc编译一个二进制的时候，默认是不加调试信息，但是不去除符号表的。这两个东西都可以辅助我们进行调试。一般来说，一个软件的release版本是不带调试信息的，逆向的软件当然也不会带调试信息。

汇编代码

  • display: Print value of expression EXP each time the program stops.

  • assemble: 这个命令可以把指令汇编成二进制

    • asm 指令1; 指令2; ......;显示汇编
    • asm -l 地址 指令1; 指令2; ......可以直接把汇编的内容写进内存

  • disassemble: 可以把打印当前函数的汇编代码，通过指定参数可以反汇编任意一段内存的汇编代码。

    • 如果要在任意情况下反汇编后面要执行的代码：

      (gdb) set disassemble-next-line on

    • 如果要在后面的代码没有源码的情况下才反汇编后面要执行的代码：

      (gdb) set disassemble-next-line auto

控制流

  1. entry-break：gef中一个强大的命令，让程序在开始处停顿。他会依次检查main,__libc_start_main,ELF中规定的Entry point。在gdb中，start是类似的命令。
  2. ni和si对应汇编级别下的单步跳过和单步进入。
  3. r,c: run/continue
  4. b 设置断点，d清除断点。可以在地址、函数、文件行打断点。
    1. break ... if cond 条件断点
    2. tbreak一次性断点
    3. hbreak可以设置硬件断点
    4. b ( ) if ( )条件断点
  5. watch: 设置内存断点，也就是当一个变量值发生变化时，程序会停下来。比如watch *(int*)0x6009c8。
  6. 对于x86，我们还可以用flags命令来修改flag寄存器，比如edit-flags [(+|-|~)FLAGNAME ...]
  7. finish: 执行完当前函数。return可以直接强制返回。

默认情况下，gdb不会进入不带调试信息的函数。可以执行“set step-mode on”命令，这样gdb就不会跳过没有调试信息的函数。

运行信息

程序本身有许多信息；程序运行到每一个点，会有很多状态可以查看。

命令作用
info all-registers查看所有的寄存器
info all-registers name查看某个寄存器
info b查看所有断点
info address sym查看符号sym的存储地址
info variables查看所有的变量（符号表和调试信息）
info functions查看所有的函数（符号表和调试信息）
info locals查看局部变量的值
display[/FMT] exp按照FMT指定格式打印exp的内容
bt打印调用栈
frame打印当前栈帧

查看内存

内存实在是太重要的，无论是数据还是指令，都是存在内存中。x 是 examine 的缩写格式，使用方法： x /[n][f][u] <addr>

  • [n]表示要显示的内存单元的个数

  • [f]表示显示方式, 可取如下值

    [f]含义
    x按十六进制格式显示变量。
    d按十进制格式显示变量。
    u按十进制格式显示无符号整型。
    o按八进制格式显示变量。
    t按二进制格式显示变量。
    a按十六进制格式显示变量。
    i指令格式
    s字符串格式
    f按浮点数格式显示变量。

  • [u]表示一个地址单元的长度, 可取如下值

    [u]含义
    b单字节
    h双字节
    w四字节
    g八字节
  1. hexdump系列， 打印内存的值，hexdump [qword|dword|word|byte][LOCATION] [[L][SIZE]][REVERSE]
  2. 比如hexdump byte $sp 10，Byte还会显示ascii码对应的字符layout

我也可以查找内存中的指定内容。比如我想找RustSBI中的mret在哪里:find 0x80000000,0x80200000,(int)0x30200073

也可以修改内存中的内容。有一个场景，是我试图修改function的return address。首先用i frame查看栈情况，知道返回地址保存在0xffffd14c处，值为0xf7df1ee5。我将他修改成0x0904853d。

Stack level 0, frame at 0xffffd150:
 eip = 0x8048557 in main; saved eip = 0xf7df1ee5
 Arglist at 0xffffd148, args: 
 Locals at 0xffffd148, Previous frame's sp is 0xffffd150
 Saved registers:
  ebp at 0xffffd148, eip at 0xffffd14c
gef➤  set {int}0xffffd14c=0x0804853d
gef➤  i frame
Stack level 0, frame at 0xffffd150:
 eip = 0x8048557 in main; saved eip = 0x804853d
 Arglist at 0xffffd148, args: 
 Locals at 0xffffd148, Previous frame's sp is 0xffffd150
 Saved registers:
  ebp at 0xffffd148, eip at 0xffffd14c

寻址方式Location

我们经常需要获得某个地址的信息，无论是查看反汇编、内存内容，或是设置断点。那么在GDB中，我们有哪些指定地址的方式呢？

  • 使用symbol table里的符号。你可以通过外部命令readelf查看，也可以用info functions和info variables查看。比如b main，disassemble task_init
  • 用绝对的数字指定地址。
    • 设置断点的时候要b *0x80000a5c，在0x80000a5c设置断点。因为直接加数字的话gdb会理解成行号。
    • 在hexdump/x的时候，hexdump qword 0x80000a5c 10查看0x80000a5c的内存，加*就跟C语言一样变成取地址的内容了
  • 利用寄存器：
    • 查看内存hexdump qword $sp 10。
    • 如果是设置断点，用rb指令。rb ra
