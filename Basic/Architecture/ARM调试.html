ARM self-hosted debug

4 Armv8-A debug

4.4 Operating system debugging

Operating system debugging允许运行在EL2的调试器程序，调试运行在EL0和EL1的代码。EL0和EL1产生的调试异常在EL2处理。Operating system debugging允许Hypervisor调试操作系统。

一些重要的管理调试的寄存器：

  • MDSCR_EL1, Monitor debug System Control Register
  • MDCR_EL2, Monitor debug Configuration Register (EL2)

6 Debug Exceptions

调试器程序作为异常处理程序代码安装，处理调试异常并且修改调试寄存器。调试器处理的调试异常是同步异常，它们被路由到调试器的异常级别。

调试器程序所处的异常级别称作debug target Exception level(ELd)，调试器程序的代码可以看作是运行在ELd的异常处理代码。ELd可能是EL1或者EL2，如果EL3使用AArch32的话也可以将调试异常路由到EL3。

当在ELd上处理调试异常时，调试器程序有下面几种办法获得异常原因

  • ESR_ELx编码的异常的种类。
  • 对于Watchpoint exception，FAR_ELx表示检测的地址。

Self-hosted调试支持下面的调试异常：

  • Breakpoint instruction(软件断点)：当PE执行BRK指令时产生。
  • Breakpoint(硬件断点)：当PE执行特定地址的指令时产生。
  • Watchpoint(内存断点)：当PE访问特定地址时产生。
  • Software step(单步执行)：当PE执行一条指令后产生。

6.1 Breakpoint Instruction Exception

6.2. Breakpoint exception

当PE访问特定地址时产生Breakpoint exception。断点对应的代码地址，存储在hardware breakpoint registers中。当PE将要执行hardware breakpoint registers存储的地址的代码时，产生Breakpoint Exception。

所以，要创建硬件断点，需要下面两步：

  1. 将断点地址存储在DBGBVR寄存器中。Debug Breakpoint Value Registers
  2. 使能断点控制寄存器DBGBCR的DBGBCR.E。Debug Breakpoint Control Registers

DBGBVR和DGBBCR有很多种设置的方式。其中一种，是让DBGBVR为要断的虚拟地址，DBGBCR.E = 1, DBGBCR.BT = 0。

  • DBGWCR0_EL1 and DBGWVR0_EL1 are for watchpoint 0
  • DBGWCR1_EL1 and DBGWVR1_EL1 are for watchpoint 1
  • DBGWCR2_EL2 and DBGWVR2_EL1 are for watchpoint 2
  • DBGWCR<n>_EL1 and DBGWVR<n>_EL1are for watchpoint n.

7 Enabling and routing for debug exception

7.1. Enabling debug exceptions for self-hosted debugging

软件断点异常总是被允许的，而且不能be masked。对于其他的调试异常，我们可以选择开启或禁用他们。下面是开启调试异常的步骤：

  1. 清除OS Lock。在AArch64上，是将OSLAR_EL1.OSLK(0)清空。
  2. 清除OS double lock，如果有实现的话。在AArch64上，是将OSDLR_EL1.DLK(0)清空。
  3. 如果调试Secure code需要清除MDCR_EL3.SDD
  4. 如果 要调试的代码 和 调试器的代码 ，运行在同一个Exception Level，将MDSCR_EL1.KDE设置成1。
  5. 如果KDE被设置了，那么需要确保 要调试的代码 的PSTATE.D被清0。可以通过MSRDAIFclr, #8指令，或者通过设置SPSR_ELd.D为0然后执行ERET，返回到要调试的代码的Exception Level来完成清0。
  6. 设置MDSCR_EL1.MDE/DBGDSCRint.MDBGen为1。他允许除了单步调试之外的调试异常。
  7. 设置MDSCR_EL1.SS为1。将允许单步调试。
    1. 如果SS被设置了，请保证PSATE.D被设置成1了。
  8. 在手册中没有提到，但是PMC应该设置为1。

7.2 Routing for debug exceptions

在ARMv8-A的异常模型中，EL0产生的异常是不能在EL0处理的，这些异常需要被路由到更高的Exception Level。EL0产生的调试异常总是被路由到EL1或EL2，这意味着调试器总是应该安装在EL1或者EL2。

如果调试器安装在EL1，将MDCR_EL2.TDE设置为0，在这种情况下，所有的调试异常都会被路由到EL1；如果调试器安装在EL1，将MDCR_EL2.TDE设置为1，在这种情况下，所有的调试异常都会被路由到EL2。
