Reticence's Notes

字符串匹配算法

字符串匹配又称模式匹配（pattern matching）。该问题可以概括为“给定字符串S和T，在主串S中寻找子串T。字符T称为模式串 (pattern)。我是一个算法小白，这里介绍的大多数算法我都只是做一个了解。如果后续有机会的话好好研究。

1模式串-1字符串

我们要解决的问题就是：给定字符串S和T，在主串S中寻找子串T。也就是一个标准的字符串匹配工作，也就是很多语言都有的str.find()函数（或类似的函数）。

单串匹配的暴力算法是显而易见的，复杂度为$O(|S||T|)$。但是实际上优化空间是很大的，下面要介绍的KMP算法就是一个复杂度为$O(|S|+|T|)$的算法。

前缀函数算法

对于一个长度为$n$的字符串$s$，他的**前缀函数$\pi(i), i\in[0,n-1]$**被定义为字符串s的子串s[0..=i]的最常的相等的真前缀和真后缀的长度。例如，对于字符串abcaba

  • $\pi(0) = 0$，因为a连真前缀都没有，他太短了。
  • $\pi(1) = 0$，ab的唯一真前缀a不等于唯一真后缀b
  • $\pi(2) = 0$，abc没有相等的真前缀和真后缀。
  • $\pi(3) = 1$，abca有相等的真前缀和真后缀a。
  • $\pi(4) = 2$，abcab有相等的真前缀ab。
  • $\pi(5) = 0$，abcabd没有相等的真前缀和真后缀。

那么，我们如何求前缀函数呢？我们可以用O(n)的时间代价算出$\pi(i)$，当然结果需要用O(n)的空间代价存储，这是废话了。程序如下

#![allow(unused)]
fn main() {
fn prefix_function(s: &[u8]) -> Vec<usize> {
    // pi存储着前缀函数的值
    let mut pi = vec![0; s.len()];
    let mut j;

    for i in 1..s.len() {
        j = pi[i - 1];
        while j > 0 && s[i] != s[j] {
            // 这个有点奇怪，换成j-=1会好理解，但是效率不高
            j = pi[j - 1];
        }
        if s[i] == s[j] {
            j += 1;
        }
        pi[i] = j;
    }
    pi
}
}

首先基于这样一个事实，那就是字符串长度+1（i+1），得到的前缀函数最多+1。所以我们从j = pi[i-1] 开始比较。

我们其实要比的真前缀和真后缀分别是 s[0..=j] 和 s[i-j..=i]，所以我们比较s[i]是否等于s[j]。如果不等的话，我们可以直接跳到j = pi[j-1]再去比较，因为我们要从之前的匹配失败中获取教训。以abcabd为例：

  1. 因为j = pi[4] = 2，也就是说之前前缀ab == ab匹配上了。我们比较c和最后一个d，发现不相等。
  2. 这个时候，我们就应该意识到，我们没有必要再比较第一个b是否等于最后一个d了，没有意义。因为我们已经发现pi[pi[4]-1] == 0，也就是说a != b，所以ab != bd。

Knuth-Morris-Pratt算法

上面对前缀函数的介绍是为了引出KMP算法。他解决的就是单串匹配问题。这个算法的复杂度是$O(|t| + |s|)$，比暴力做法$O(|t||s|)$好多了。

这个算法的思路是，构造一个新的字符串s+#+t，其中#为一个既不出现在s也不出现在t的分隔符。然后计算该字符串的前缀函数。

我们考虑得到的前缀函数pi的意义，从pi[|s| +1 .. |s| + |t| + 1]，如果其中有一个等于n，意味着什么？意味着我们在t找到了一个和s相同的子串。分隔符存在的意义是让pi[x]不会大于n。

当然，在实现的时候，你并不需要真的构造这个s+#+t。

Boyer-Moore算法

BM算法也是解决单串匹配问题的。他的思路和KMP算法有所不同。它采用的是后缀的匹配，而KMP是前缀。

不过也是利用之前匹配失败的信息减少匹配次数。他的时间复杂度和KMP是一样的。不过，BM算法更适合用来解决自然语言的匹配问题，而KMP算法更适合用来解决匹配串小的匹配问题，比如DNA。1

Boyer-Moore-Sunday算法

Sunday算法通常用作一般情况下实现最简单而且平均表现最好之一的实用算法，通常表现比BM要快一点。

BMHBNFS 算法

这个算法根据时间节省还是空间节省为第一目标，会有差别巨大的不同实现。空间节省的版本被CPython用来实现字符串的查找2。

1模式串-n字符串

先考虑一个简单的问题，是1个模式串t是否在n个字符串S中。你可以用暴力的方法一个一个进行匹配，时间代价为$O(n|t|)$（注意这里不是判断子串，而是相等）。不过你可以想到这个问题就像查字典一样，你要在字典S中找单词t是不可能一个一个找过来的，肯定是先找第一个字母，再找第二个字母这样。所以我们引入了字典树来解决这个问题。

然后如果问题是判断1个模式串是否是n个字符串中某一个的子串，一种方法将它们直接连起来便可作为一个待匹配串处理（串之间可以插入不可能出现的字符，防止如模式串为'abc'，匹配上了'aaa'和'bcd'的情况），转换成1模式串-1字符串问题用KMP算法解决，那么时间代价为$O(n|s| + |t|)$；或者直接用n次KMP算法，这个效率会稍微低一点，为$O(n(|s| + |t|))$。

Trie字典树

字典树用于统计、排序、和保存大量字符串。所以经常被搜索引擎系统用于文本词频统计、自动补全。它的优点是：利用字符串的公共前缀来减少查询时间，最大限度地减少无谓的字符串比较。如下图，保存了to, tea, ted, ten, A, i, in, inn。

用于查找：顺着树遍历下来就好了，时间代价变成$O(|t|)$。

用于自动补全：比如上面这个例子，你输入了t之后他就可以提示你to/te。

基本的数据结构可以这么表达。当然可以给节点添加其他的属性。使用很简单，构建一棵树，找的时候顺着树搜索就可以了。

#![allow(unused)]
fn main() {
struct Node { 
    children: [Option<Box<Node>>; 26],
    is_word: bool,
}
}

还可以使用double array trie，即用两个数组记录转移目标。double array trie（简称DAT，也有人简称DART）包括base和check两个数组，其中base中记录结点转移表的基地址，check记录由哪个结点转移过来。

哈希表

如果只是为了判断t是否在S中，也可以用哈希表。因为很多语言都内置了“集合”这个数据结构，所以会非常方便。理想情况的哈希表的查找时间复杂度为$O(1)$，不过他只能解决精确匹配的情况，没有Trie那么强大。

这种问题叫做“多模匹配”问题。

自动机

下面的自动机指的是“确定有限状态自动机DFA”，没有学过算法但是大学学过编译原理的同学回想起来这玩意儿在词法分析的时候学过。自动机接受一个字符串，然后给出匹配/不匹配的结果。所以我们上面的KMP算法、Trie树都可以算作是自动机。

AC自动机

AC自动机是一种多模匹配算法。简单来说，建立一个 AC 自动机有两个步骤：

  1. 将所有的字符串$S$构成一棵 Trie。按照之前的办法构建。
  2. KMP 的思想：对 Trie 树上所有的结点构造失配指针。

B站的这个可视化视频简直太妙了。失配指针，目的是为了让匹配失败的时候，去尝试匹配其他的节点。

后缀自动机

后缀自动机(Suffix Automation)也可以解决多模匹配问题，后缀自动机几乎能用于所有与子串相关的问题，比如寻找两个字符串的公共子串。

他的基本思路可以认为是在Trie上存储了一个字符串的所有子串。3

如果alphabet的字符数量为k，字符串长度为n，那么后缀自动机的时间复杂度和空间复杂度都是$O(n logk)$，一般我们认为k是常数，就是$O(n)$。4
