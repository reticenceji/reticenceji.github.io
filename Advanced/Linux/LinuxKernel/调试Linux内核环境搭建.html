Reticence's Notes

调试Linux内核环境搭建

下面介绍的调试的总体思路有三个，需要的环境是一个Linux环境，最好是主机，虚拟机也行。假设你已经知道了gdb调试的基本原理。

  1. 编译内核，用Qemu启动，用gdb调试。这还需要制作rootfs。这个需要过程(1,2,4)
  2. 编译内核，用VMware运行，用gdb调试。这个需要过程(6,7)，了解过程(1)。
  3. 下载已安装的内核版本的带调试信息的镜像，用gdb调试。这个需要过程(6,7)。
sudo apt install qemu-system-x86 gdb debootstrap\
	libncurses5-dev openssl libssl-dev build-essential pkg-config libc6-dev \
	bison flex libelf-dev zlibc minizip libidn11-dev libidn11

1. 内核编译

首先，你需要下载内核的源代码。比如在https://kernel.org中下载。你可以选择下载[tarball]，然后使用tar -xfv xxx.tar.xz命令进行解压。解压后进入源代码目录。

推荐使用下面的命令进行编译配置，使用字符图形界面来对整个系统进行裁剪。也有其他的选项，不过这个就挺好的。

make menuconfig
  1. 开启调试选项: Kernel hacking -> Compile time checks and compiler options -> Compile kernel with debug info。这个默认好像就是开启的。
  2. warn for stack frames larger than 的数值，将其修改为4096（最好不要大过这个数值）
  3. 配置完成之后，保存在默认的.config文件就好了。需要注意的是，请在ext4文件系统下进行编译。否则会有意想不到的错误。

然后进行编译，这需要一点时间。

make -j$(nproc)

我遇到的一些编译错误的处理，其实谷歌都会告诉你。

  • 如果遇到报错make[1]: *** No rule to make target 'debian/canonical-certs.pem', needed by 'certs/x509_certificate_list'. Stop.，编辑.config文件，修改CONFIG_SYSTEM_TRUSTED_KEYS，将其置空，重新make即可。
  • 如果遇到报错make[1]: *** No rule to make target 'debian/canonical-revoked-certs.pem', needed by 'certs/x509_revocation_list'. Stop.可以执行scripts/config --disable SYSTEM_REVOCATION_KEYS

2. 制作rootfs

系统的启动是需要挂载rootfs的。我们使用debootstrap进行制作。

qemu-img create rootfs.img 1G
mkfs.ext4 rootfs.img
# 切换到随便一个目录，/tmp也行
mkdir mnt
# 挂载image到mnt
sudo mount -o loop rootfs.img ./mnt/
sudo debootstrap bionic ./mnt/ #18.04
sudo debootstrap focal ./mnt/  #20.04
# 挂载一些必要的文件系统
sudo mount -t proc /proc ./mnt/proc
sudo mount --rbind /dev ./mnt/dev
sudo mount --rbind /sys ./mnt/sys
# 如果稍后安装软件的时候提示证书错误，把source.list里面的https改成http
sudo cp /etc/resolv.conf ./mnt/etc/resolv.conf
sudo cp /etc/apt/source.list ./mnt/etc/apt/source.list

sudo chroot ./mnt # 切换根目录
passwd root       # 修改root密码
apt update        # 看情况安装你需要的软件
apt install vim gcc make build-essential 

# 卸载image
sudo umount mnt
  • 文件系统是只读的，可以使用mount -o remount /dev/sda /修改成可读写的。
  • 至今不知道Qemu中如何上网，只能从上面标记的部分挂载到主机上下载软件了。
  • 如果要移动文件，只需要把他挂载到主机就可以了。
  • fsck.ext4 -y /dev/sda，有时你把盘搞坏了可以修复一下。

3. 运行

使用qemu运行。

# 启动qemu虚拟机
sudo qemu-system-x86_64 \
     -M pc \
     -hda rootfs.img \ # 制作的rootfs
     -kernel bzImage \ # 编译产生的Image, 在/boot/x64里面
     -append "root=/dev/sda console=ttyS0" \
     -nographic

4. 调试

使用qemu+gdb进行调试。调试的时候添加选项-append nokaslr，这样子内核的布局就会和vmlinux一样，调试会方便很多很多很多。

# 启动qemu虚拟机
sudo qemu-system-x86_64 \
     -M pc \
     -hda rootfs.img \
     -kernel bzImage \
     -append "root=/dev/sda console=ttyS0 nokaslr" \
     -nographic \
     -s -S
     
# 另一个例子
sudo qemu-system-aarch64 -M virt \
    -cpu max \
    -smp 2 \
    -m 512 \
    -kernel ./Image \
    -nographic \
    -append "console=ttyAMA0 root=/dev/ram rdinit=/sbin/init nokaslr" \ # 关闭 KASLR
    -initrd ./rootfs.cpio.gz\
        -fsdev local,security_model=passthrough,id=fsdev0,path=./share\
    -device virtio-9p-pci,id=fs0,fsdev=fsdev0,mount_tag=hostshare \
    -s

启动gdb

$ gdb
(gdb) file vmlinux
(gdb) target remote localhost:1234

5. 跨平台

有时候，你可能需要跨平台编译和调试。这里假设环境是x64的Ubuntu，想要编译arm64的内核。首先安装跨平台的工具

sudo apt install qemu-system-arm \        # qemu
         binutils-aarch64-linux-gnu \     # objdump等二进制工具
         gcc-8-aarch64-linux-gnu \        # gcc
         g++-8-aarch64-linux-gnu \        # g++ 这个编译内核并不是必须的
         gdb-multiarch                    # 多平台gdb

然后编译内核

export ARCH=arm64
make CROSS_COMPILE=aarch64-linux-gnu- defconfig
make CROSS_COMPILE=aarch64-linux-gnu- menuconfig  # 启用 PA 支持（默认已开启）
make CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)

6. 在虚拟机中运行自己编译的内核

首先，你需要有一个虚拟机软件，比如VMware/VirtualBox。然后在上面跑着一个Linux系统，比如Ubuntu。接下去我们要做的是在上面编译内核，然后替换内核。

在前面源代码的基础上，我们复制主机的编译配置文件到源代码目录。之前提到通过make menuconfig进行编译配置，现在就不用了，不过也可以执行一下，进行配置。

cd linux # 进入源代码目录
cp -v /boot/config-$(uname -r) .config

在虚拟机中当然也要安装必要的编译工具，在文章的开头有提到。

make -j $(nproc) # 编译内核
sudo make modules_install # 安装模块
sudo make install # 安装内核

别忘了把编译生成的vmlinux文件拿出来，调试用。

更新grub config，重启。

sudo update-initramfs -c -k 5.16.9
sudo update-grub
reboot

另外，如果你只是为了学习一下Linux，而不是自己修改了一些文件必须重新编译内核，你也可以选择下载别人的编译结果，这非常方便。参考这篇Ubuntu Wiki。

首先，我们通过下面的命令创建/etc/apt/sources.list.d/ddebs.list，更新列表。

echo "deb http://ddebs.ubuntu.com $(lsb_release -cs) main restricted universe multiverse
deb http://ddebs.ubuntu.com $(lsb_release -cs)-updates main restricted universe multiverse
deb http://ddebs.ubuntu.com $(lsb_release -cs)-proposed main restricted universe multiverse" | \
sudo tee -a /etc/apt/sources.list.d/ddebs.list
sudo apt install ubuntu-dbgsym-keyring
sudo apt-get update

然后可以下载带有调试信息的image。

sudo apt-get install linux-image-`uname -r`-dbgsym

vmlinux在/usr/lib/debug/boot/vmlinux-4.15.0-45-generic，把他拿出来就可以调试了。

7. 利用VMWare进行调试

修改虚拟机的vmx文件，添加下面这一行。显然，这是针对64位系统的。

debugStub.listen.guest64 = "TRUE"

另外，我们应该关掉KASLR，否则调试会非常痛苦：修改/etc/default/grub文件，找到GRUB_CMDLINE_LINUX_DEFAULT然后添加在最后面添加nokaslr。然后更新grub，sudo update-grub。

然后当VM开机的时候8864端口就会开启，我们可以用gdb连接这个端口进行调试。

$ gdb
(gdb) file vmlinux
(gdb) target remote localhost:8864
